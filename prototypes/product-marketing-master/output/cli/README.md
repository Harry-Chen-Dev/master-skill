## 用法

所有脚本支持 `--help` / `--explain` / `--dry-run` / `--json` 四个标准 flag。

```bash
# 拿到新问题时, 按 N 个研究维度做功课
./protocol/agentic.sh

# 决策树评估 (基于 playbook)
./decision/topic-1.sh
# SOP 走查 (workflow)
./workflow/positioning-april-dunford-10-ste.sh

# 看背后的心智模型 / playbook (不交互)
./protocol/agentic.sh --explain
```

| 脚本 | 作用 |

| `protocol/agentic.sh` | Agentic Protocol (7 维度) — 拿到新问题时按这一行的研究维度做功课 |

| `decision/topic-3.sh` | 一个 决策树 (3 条规则) |

| `decision/topic-5.sh` | 竞争 决策树 (1 条规则) |
| `workflow/positioning-april-dunford-10-ste.sh` | 给产品做定位 (Positioning — April Dunford 10-step) SOP 走查 |
| `workflow/messaging-messaging-house.sh` | 写核心 messaging (Messaging house) SOP 走查 |
| `workflow/product-launch.sh` | 跑一次产品发布 (Product launch) SOP 走查 |
| `workflow/battlecard-competitive-analysis-.sh` | 做竞品分析 + 出 battlecard (Competitive analysis & battlecards) SOP 走查 |
| `workflow/win-loss-win-loss-analysis.sh` | 跑 win/loss 分析 (Win/loss analysis) SOP 走查 |
| `workflow/icp-buyer-persona-icp-buyer-pers.sh` | 建 ICP + buyer persona (ICP & buyer persona) SOP 走查 |
| `workflow/messaging-message-testing-with-r.sh` | 用真实买家测 messaging (Message testing with real buyers) SOP 走查 |
