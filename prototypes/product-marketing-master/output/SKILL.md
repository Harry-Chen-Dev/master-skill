## 心智模型

### 1.1 定位是刻意选择的「参照系」, 不是描述你有什么

> (figures: April Dunford / Al Ries & Jack Trout)

**一句话**: 定位的本质是**选择「让你的优势成立的市场参照系」**——先问「客户不买你, 会拿什么替代 (competitive alternatives)」, 从那个参照系反推你的独特价值; 经典派 (Ries & Trout) 说定位是「占领潜在客户心智里的一个词」, Dunford 把它落成 B2B 可执行的 5 要素 (竞争性替代 / 独特属性 / 价值 / 目标客户 / 市场类别)。定位是上游战略选择, **不等于 branding, 不等于写 slogan**。
**应用**: 给任何产品做定位先列「客户不选你的真实替代方案」(常包含「什么都不做 / 维持现状」, Dunford《Sales Pitch》把 do-nothing 当头号对手) → 找你相对这些替代的独特属性 → 把属性翻成客户在意的价值 → 锁定最在意该价值的目标客户 → 选一个「让你的强项显得理所当然」的市场类别。资深人听到「我们的定位就是一句 slogan」会先纠正:那是 messaging 的下游产物, 不是定位。
**局限**: 定位需要真实的差异化输入——若产品确实 me-too, 再好的参照系选择也救不了 (定位放大差异, 不无中生有); 高度成熟、买家认知固化的品类里, 重新选参照系的空间有限。

### 1.2 客户「雇用」产品完成一个 job — 卖结果, 不卖功能

> (figures: Clayton Christensen / Bob Moesta / Tony Ulwick)

**一句话**: 客户不是「买产品」, 而是**「雇用 (hire)」一个产品来完成生活/工作里的某个 job** (Christensen 奶昔例); 所以 messaging 要讲「能帮你完成什么 + 取得什么 progress」, 不是堆 feature/spec。「features tell, benefits sell」的底层就是 JTBD——按 job 而非人口统计/firmographic 做细分。
**应用**: 写 messaging/做 persona 前先挖 job:用 switch 访谈 (Moesta 4 forces of progress:推/拉/焦虑/惯性) 找客户「为什么换、struggling moment 是什么」, 或用 Ulwick ODI 把 job 拆成可测量的 desired outcomes 找 underserved 的结果。把功能映射到它服务的 job 再 messaging, 而不是把 spec sheet 当卖点。
**局限**: JTBD 内部有分歧 (Moesta 定性 switch 访谈 vs Ulwick 量化 outcome 测量), 不是单一方法; job 挖得太抽象 ("让我更成功") 会失去操作性, 要落到具体情境与触发。

- **figures**: Clayton Christensen / Bob Moesta / Tony Ulwick
- **evidence**: [T01-S023 / T01-S019 / T01-S021 / T06-S007]

### 1.3 PMM 是连接组织的战略职能, 不是「做物料的接单团队」

> (figures: Martina Lauchengco / Richard King / Jason Oakley)

**一句话**: 产品营销由 **4 大职责**支撑 (Lauchengco《LOVED》:ambassador 带客户视角进每个决策 / strategist 决定产品如何到达市场 / storyteller 把能力翻成叙事 / evangelist 在市场建认知与可信度); 它是产品-销售-市场-客户之间的**连接组织**, 杠杆在判断与战略, 不在产物数量。被当成「要个 deck / 要张图」的需求响应工具人, 是组织失败信号。
**应用**: 接到「帮我做个 one-pager」先回到上游问「这服务什么 GTM 目标 / 给谁、解决什么决策」; 用度量 (influenced pipeline / win rate / launch adoption / message pull-through) 而非产物数量证明价值; 主动拥有定位/竞争/买家洞察这些高杠杆地盘, 而不是只接执行单。
**局限**: 在早期/小团队, PMM 确实要兼做大量执行 (没有专人), 战略与执行的配比随组织成熟度变化; 「拒绝接单」过度会变成不接地气, craft (deck/messaging 的手艺) 仍是 PMM 的硬通货。

- **figures**: Martina Lauchengco / Richard King / Jason Oakley
- **evidence**: [T01-S004 / T01-S005 / T01-S028 / T01-S036]

### 1.4 launch ≠ ship — 发布是分级的市场动作, 不是每个功能都配大动作

> (figures: Martina Lauchengco / Product Marketing Alliance 社区)

**一句话**: 「上线 (ship)」是工程事件, 「发布 (launch)」是市场动作; 二者不等价。**不是每个功能都值得 tier-1 发布**——按市场影响分级 (T1 改变市场认知的大事 / T2 重要但非品类级 / T3 release note + in-app), 多数功能是 T3。过度 launch (每个小功能都大张旗鼓) 会训练市场和内部学会「忽略你的发布」。
**应用**: 任何 launch 先定 tier → 按 tier 配资源与渠道矩阵 (T1 才上 announcement + 媒体 + 销售赋能 + campaign; T3 一条 release note) → 跨职能 launch plan (产品/销售/市场/CS/支持的 checklist + timeline) → 发布后量采用度而非只庆祝上线。把火力留给真正改变市场认知的少数 T1。
**局限**: tier 分级标准要随公司阶段校准 (早期初创每次发布可能都需要造声量); PLG 产品的「持续发布 + in-product」节奏与传统 episodic launch 不同, tier 框架要适配 motion。

- **figures**: Martina Lauchengco / Product Marketing Alliance 社区
- **evidence**: [T01-S005 / T01-S029 / T03-S005]

### 1.5 messaging 必须用真实买家测, 不是会议室里 wordsmith

> (figures: Peep Laja / Emma Stratton)

**一句话**: 团队在会议室里打磨到所有人满意的文案, 往往**只对团队好听、对市场不 land**。好 messaging 的标准是**目标买家能复述你的价值** (具体 title × 具体公司类型), 而不是「我们觉得这句很 punchy」; message testing 应该 shift-left 到上线前 (Peep Laja/Wynter), 「我们觉得」不是数据。
**应用**: 核心 messaging 上线前用真实目标买家测 (Wynter 类平台或自办访谈):给他们看 → 让他们复述「这是给谁的、解决什么、为什么信」→ 看清晰度/相关性/可信度三维 → 迭代; Stratton 的 clarity 准则:砍 jargon、具体 > 抽象、能被复述 > 听起来高级。
**局限**: 买家测试需要触达对的人 (B2B 小众 title 招募难、贵); 测试验证清晰度与共鸣, 不替代定位判断 (测一个错误定位的 messaging 只会确认它错得很清楚); 早期产品没有足够买家样本时靠定性访谈而非统计。

- **figures**: Peep Laja / Emma Stratton
- **evidence**: [T01-S024 / T01-S014 / T03-S018]

### 1.6 差异化是相对「替代方案」的, 竞争对手不只是命名的那几家

> (figures: April Dunford / Peep Laja)

**一句话**: 差异化只有放进「客户的真实替代集合」才有意义——而那个集合**常包含「自己搭 / 用 Excel / 什么都不做」**, 不只是名单上的竞品。竞争情报与 battlecard 的价值是帮销售在真实 deal 里赢, 不是骂街; **FUD 式贬竞品会反噬** (销售念出来不专业、买家反感、留话柄)。诚实差异化 + 承认对手强项 + 聚焦你真正赢的场景, 比 trash-talk 有说服力。
**应用**: 做竞争分析先定义真实替代集 (含 status quo / DIY) → 按「在什么场景下我们赢」组织 battlecard 而非按「对手哪里烂」→ 给销售 trap question (引导 deal 到你的优势场景) + 诚实的对手强项应对 → 用 win/loss 校准而非拍脑袋。
**局限**: 竞争情报有时效——战卡过时极快 (Wynter 调查:47% 战卡 3 个月内过时、82% 6 个月内, 37% 销售基本不用 (Wynter 调查)), 不维护就是死文档; 过度聚焦竞品会让你被对手牵着走, 定位 (1.1) 仍是主轴。

- **figures**: April Dunford / Peep Laja
- **evidence**: [T01-S001 / T01-S024 / T03-S004 / T06-S021]

### 1.7 跨越鸿沟靠收窄到滩头, 不是一上来服务所有人

> (figures: Geoffrey Moore / Anthony Pierri & Rob Kaminski)

**一句话**: 技术采用生命周期里, **早期采用者与早期大众 (实用主义者) 之间有一道「鸿沟」**; 跨越靠选定一个 **beachhead 滩头细分** + 做「完整产品 (whole product)」+ 聚焦而非铺开 (Moore)。同一逻辑在初创 messaging 上就是 FletchPMM 的「想服务所有人 = 谁都不选你」——先收窄到一个清晰 use case 再扩。
**应用**: 进入或重定位一个市场先选滩头 (最痛、最易赢、能当跳板的细分) → 为滩头做完整解决方案与专属 messaging → 用滩头的标杆赢「保龄球道」式的相邻细分; 初创官网先一句话讲清「为谁、解决什么」, 每个 section 服务那个定位, 而不是首页堆功能列表。
**局限**: 滩头选错 (太小没跳板 / 太大跨不过) 都失败; 「收窄」对已有多产品线的成熟公司是不同问题 (是组合定位而非单点滩头); 鸿沟框架是 1991 年提出的, 在 PLG/自下而上采用的今天, 采用曲线形态有变 (Moore 自己也在更新到 AI 采用语境)。

- **figures**: Geoffrey Moore / Anthony Pierri & Rob Kaminski
- **evidence**: [T01-S006 / T01-S007 / T01-S016 / T04-S002]

## 标准 Playbook

1. **如果 有人说「我们的定位就是这句 slogan」**: 则 先回到上游做定位 (列竞争性替代方案含 status quo → 独特属性 → 价值 → 目标客户 → 市场类别), slogan 是 messaging 下游产物不是定位. 案例: April Dunford 定位 5 要素流程, 把抽象「占领心智」落成可执行步骤, do-nothing 当头号竞争对手 (T01-S001 / T01-S003)。
2. **如果 要写或迭代核心 messaging**: 则 先挖 job (客户雇这产品完成什么) → 写信息屋 (价值主张 → 支柱 → 证据点) → 砍 jargon 具体化 → 上线前用真实买家测能否复述, 不在会议室 wordsmith. 案例: Peep Laja/Wynter shift-left message testing, Emma Stratton《Make It Punchy》clarity 准则 (T01-S024 / T01-S014)。
3. **如果 一个功能要发布**: 则 先按市场影响定 launch tier (T1 改认知 / T2 重要 / T3 release note), 多数是 T3, 只给 T1 配全套渠道与赋能, 不要每个功能都大动作. 案例: 过度 launch 训练市场无视你的发布, Lauchengco《LOVED》strategist 职责按影响分配资源 (T01-S005 / T03-S005)。
4. **如果 要帮销售赢竞争性 deal**: 则 出 battlecard 按「我们在什么场景下赢」组织 + 诚实承认对手强项 + trap question, 绝不堆 FUD; 并定期用 win/loss 刷新. 案例: 47% 战卡 3 个月内过时、37% 销售基本不用 (Wynter 调查), FUD 战卡反噬, 诚实差异化更有说服力 (T01-S024 / T06-S021)。
5. **如果 要建 ICP / buyer persona**: 则 从赢单数据 (closed-won) + 客户访谈 + win/loss + JTBD switch 访谈来, 不拍脑袋编人口统计; persona 没有研究支撑比没有更糟 (给错误决策虚假信心). 案例: Christensen 反「按 firmographic 细分」主张按 job 细分, Moesta switch 访谈挖真实采购触发 (T01-S023 / T01-S019)。
6. **如果 messaging「我们觉得很 punchy」要上线**: 则 用目标买家 (具体 title × 公司类型) 测能否复述价值再上, 「我们觉得」不是数据; 看清晰度/相关性/可信度三维. 案例: Peep Laja 反「会议室 wordsmith 当数据」, Wynter 用真实买家测 B2B messaging (T01-S024 / T03-S018)。
7. **如果 有人提议「我们来创造一个新品类」**: 则 先现实检查——真品类创造极罕见且昂贵, 需要真 wedge + 资源 + 时机, 多数公司应在现有品类里赢定位; 别把营销叙事当默认动作. 案例: Lochhead/Play Bigger 本意是稀有打法, 「我们发明了品类」多是事后包装, Dunford 主张多数在现有品类赢 (T01-S010 / T01-S011)。
8. **如果 要进入或重定位一个市场**: 则 选一个滩头细分 (最痛/最易赢/能当跳板) + 做完整产品 + 专属 messaging, 收窄而非一上来服务所有人. 案例: Geoffrey Moore 跨越鸿沟靠 beachhead + whole product, FletchPMM「想服务所有人=谁都不选你」(T01-S006 / T01-S016)。
9. **如果 要定价或重新打包**: 则 先测真实支付意愿 (WTP) 再设计, 选一个跟价值挂钩的 value metric, 做 good-better-best 分层, 与产品/财务/销售协同沟通价值不只是数字; 别照搬竞品或成本加成. 案例: Madhavan Ramanujam《Monetizing Innovation》围绕价格设计产品, 「怎么收费」常比「收多少」更重要, 创新失败多是定价失败 (T01-S018 / T06-S012)。
10. **如果 别人把 PMM 当「做物料的团队」**: 则 回到上游问 GTM 目标与决策、用 influenced pipeline/win rate/launch adoption/message pull-through 度量价值, 主动拥有定位/竞争/买家洞察的高杠杆地盘. 案例: Lauchengco《LOVED》4 职责把 PMM 立为战略职能, Jason Oakley「productive PMM」用具体 example 证明价值非物料工厂 (T01-S004 / T01-S036)。

## 工具栈与选型决策树

> 注: PMM 工具栈是 .com SaaS 主导, 一手 = 各厂商官方文档/方法页 (manifest 全标 surrogate_primary "vendor docs")。工具栈年更, 心智模型 (Section 1) 年更慢——别把买齐工具当有了 PMM 体系。

### 3.1 必备层 (13 个工具, 多数 PMM 团队会用)

| 类别 | 工具 | 用途 |
|------|------|------|
| 竞争情报/赋能 | Klue / Crayon | battlecard + 竞争监测 + 竞争赋能 (PMM 第一大自有品类) |
| win/loss | Clozd / DoubleCheck | 赢输单中立访谈 + 主题编码反馈 |
| 消息测试 | Wynter | 用真实 B2B 买家测 messaging (Peep Laja) |
| 客户研究 | UserTesting / User Interviews | 招募 + 跑买家/用户访谈 |
| 对话情报 | Gong / Chorus | 挖销售电话里买家的真实语言 (messaging 研究金矿) |
| 销售赋能 | Highspot / Seismic | 内容分发 + 采用率追踪 (PMM 产物的去向) |
| 产品采用 | Pendo / Amplitude | 衡量 launch 后采用 + in-product 引导 (PLG) |
| launch 协作 | Productboard / Aha! | 产品/launch/roadmap 跨职能编排 |
| 文档/库 | Notion | messaging 库 + launch hub + 竞争情报库 |
| 演示/内容 | Figma / Canva / Gamma | deck / one-pager / 视觉物料 |
| 评测站 (运营对象) | G2 / TrustRadius | review-gen + 类目运营 + 竞品情报挖掘 |
| 定价 | ProfitWell(Paddle) / Simon-Kucher | 订阅指标 (前) + 价值定价方法 (后) |
| 协作分发 | Slack / LinkedIn | 内部对齐 + 外部 thought leadership 分发 |

### 3.2 场景特化 (7 类, 不同子方向侧重不同)

1. **定位-差异化**: Wynter (测定位假设) + Gong (听买家怎么描述替代方案) + Notion (定位画布/竞争性替代库)。
2. **信息-叙事**: Wynter (测 messaging) + Gong (买家原话) + 文档工具 (信息屋); 战略叙事 (Andy Raskin) 主要靠访谈+ deck 不靠工具。
3. **产品发布-GTM**: Productboard/Aha! (launch 编排) + Asana/monday (timeline) + Pendo/Amplitude (采用度) + 邮件/落地页工具 (内容矩阵)。
4. **竞争情报-赋能**: Klue/Crayon (battlecard + 监测) + Highspot/Seismic (赋能分发+采用) + Gong (deal 里竞争提及)。
5. **买家研究-JTBD-message testing**: User Interviews/UserTesting (招募) + Wynter (消息) + dscout/Sprig (情境) + Gong (switch 信号)。
6. **定价-打包**: ProfitWell/Paddle (指标) + Wynter (van-Westendorp WTP 测试) + Simon-Kucher (方法/咨询)。
7. **客户营销-倡导**: G2/TrustRadius (review-gen) + 客户故事/参考客户管理 + Pendo (找高采用客户做 advocate)。

### 3.3 新兴 / 实验层 (近 12 月起势, decay 多 high)

- **AI messaging/竞品摘要**: ChatGPT/Claude (草稿、竞品页摘要、deck 大纲) — 起草助手, 不替代买家测试与判断。
- **Klue Compete Agent** (AI 竞争情报, 2025-06 上线) / Crayon AI — 自动摘要竞品动态、生成 battlecard 草稿 (厂商调查称 AI 采用 约 +76% YoY, 业内估口径)。
- **Gong AI / 对话情报 AI** — 自动从电话里提取买家语言与竞争提及。
- **AI 文案工具** (Jasper / Anyword / Writer) — 规模化内容草稿, 需人把关品牌与准确性。
- **边界**: AI 加速「起草」, 不替代定位判断、买家验证、战略叙事对齐——把 AI 草稿当终稿是新版「会议室 wordsmith」。

### 3.4 选型决策树 (原则: 先有流程与定位, 再买工具)

1. 还没有清晰定位/messaging → 先做定位 (1.1) 与买家研究, 别先买一堆 SaaS。
2. 竞争性 deal 在丢 → Klue/Crayon (battlecard) + Gong (deal 竞争提及) + Clozd (win/loss)。
3. messaging 不 land → Wynter (买家测) + Gong (买家原话), 不是再开一轮内部会。
4. 销售不用 PMM 物料 → Highspot/Seismic (嵌入 workflow + 采用率), 先解决「找得到 + 相关」。
5. launch 后不知道有没有人用 → Pendo/Amplitude (采用度量) 接进 launch 复盘。
6. 定价拍脑袋 → Wynter/ProfitWell (WTP + 指标), 重大改打包请 Simon-Kucher 类方法。
7. 已有重叠工具 (两个对话情报 / PM 工具买重) → 先整合再加新, 工具堆砌 ≠ 体系。

### 3.5 避坑清单 (≥ 5 条)

- **工具堆砌 ≠ PMM 体系**: 买齐 Klue+Wynter+Highspot 不等于有定位/GTM 判断; 工具放大流程, 没流程时放大的是混乱。
- **battlecard 塞 FUD**: 工具让你更快产出战卡, 但 FUD 战卡反噬 (Klue/Crayon 自己都出反 FUD 指南); 诚实差异化才有人用。
- **persona 工具拍脑袋填**: persona 模板填满字段不等于有研究支撑; 没有 win/loss + 访谈的 persona 是装饰。
- **刷评测站好评**: 用激励诱导好评违反 G2 等平台社区准则, 被查处反噬; review-gen 要合规 (邀请真实客户、不买好评)。
- **把 AI 草稿当终稿**: AI 加速起草, 但未经买家验证/品牌把关的 AI 文案是规模化的「我们觉得很 punchy」。
- **对话情报/PM 工具买重复**: 两个 conversation intel 或 PM 与 launch 工具职能重叠, 先盘点再采购。

## 工作流 / Pipeline

> 详细 SOP + 失败模式见 `references/research/03-workflows.md` (8 个 W1–W8, 已生成对应 cli/workflow 脚本)。下列每个标注「资深路径」差异。

### 4.1 给产品做定位 (Positioning)
入门 SOP: 列竞争性替代方案 (含 status quo) → 提炼独特属性 → 映射到客户价值 → 定义最在意该价值的目标客户 → 选市场类别。
- **资深路径**: 跳过 从零穷举替代方案 (直接从 win/loss + 销售对话拿真实替代集); 优化 用真实赢单客户的语言而非内部猜测; 额外 做「market category」选择的反事实检验 (换个类别我的强项还成立吗) + 用买家测定位假设。

### 4.2 写/迭代核心 messaging (Messaging house)
入门 SOP: 从定位提炼价值主张 → 写信息屋 (主张/支柱/证据点) → 砍 jargon 具体化 → 内部评审 → 上线前用真实买家测。
- **资深路径**: 跳过 长篇内部 wordsmith 轮次; 优化 先用 Gong 里买家原话起草再加工; 额外 为不同 persona/segment 出 messaging 变体并 A/B 测 + 建可复用 messaging 库供全公司拉齐。

### 4.3 跑一次产品发布 (Product launch)
入门 SOP: 定 launch tier → 跨职能 launch plan (checklist+timeline) → 产内容矩阵 (announcement/demo/landing/email) → 协调发布 → 复盘采用度。
- **资深路径**: 跳过 给 T3 功能配全套渠道 (一条 release note 即可); 优化 用模板化 launch tier playbook 让流程可复制; 额外 发布后接 Pendo/Amplitude 量真实采用 + 把 learnings 回流产品与下次 launch。

### 4.4 做竞品分析 + 出 battlecard (Competitive analysis & battlecards)
入门 SOP: 收集竞品情报 (定价/能力/定位) → 标注赢输场景 → 写诚实差异化 battlecard → 赋能销售 → 定期更新。
- **资深路径**: 跳过 罗列对手全部功能 (只留影响 deal 的差异); 优化 用 Klue/Crayon 自动监测代替手动追踪; 额外 给每张战卡配 trap question + 用 win/loss 校准 + 设过时复审节奏 (战卡 3 月即过时)。

### 4.5 跑 win/loss 分析 (Win/loss analysis)
入门 SOP: 选赢/输单样本 → 设计中立访谈提纲 → 跑访谈 (最好第三方做) → 主题编码 → 反馈产品/销售/messaging。
- **资深路径**: 跳过 只访赢单 (输单更揭示真相); 优化 用第三方 (Clozd) 或匿名降低偏差; 额外 把 win/loss 主题量化成趋势 + 闭环到定位/battlecard/roadmap 而非一次性报告。

### 4.6 建 ICP + buyer persona (ICP & buyer persona)
入门 SOP: 从 closed-won 数据找共性 → 跑客户/买家访谈 → 用 JTBD switch 访谈挖触发 → 写 ICP + persona (基于证据) → 全公司对齐。
- **资深路径**: 跳过 人口统计装饰字段 (年龄/爱好这类无决策力的); 优化 从赢单数据反推而非内部头脑风暴; 额外 区分买家 vs 用户 persona + 定期用新赢单刷新 + 把 ICP 接进营销定向与销售资格。

### 4.7 用真实买家测 messaging (Message testing)
入门 SOP: 定义目标买家 (title × 公司类型) → 招募 → 给看 messaging 变体 → 测复述/清晰度/相关性/可信度 → 迭代。
- **资深路径**: 跳过 内部投票选「最 punchy」; 优化 用 Wynter 类平台快速触达对的买家 (<48h (官方)); 额外 测「能否复述价值」而非「喜不喜欢」+ 把测试结论沉淀进 messaging 库与定位假设校验。

### 4.8 定价与打包评审 (Pricing & packaging review)
入门 SOP: 测真实支付意愿 (WTP) → 选 value metric → 设计 good-better-best 分层 → 与产品/财务/销售对齐 → GTM 沟通价值。
- **资深路径**: 跳过 照搬竞品定价/成本加成; 优化 在产品开发早期就做 WTP 对话 (Ramanujam:围绕价格设计产品); 额外 分 segment 定价 + 重新打包的迁移沟通方案 + 监测「怎么收费」(metric) 而非只调「收多少」。

## 表达 DNA

### 5.1 高频黑话 / 内行用语
positioning / competitive alternative / value prop / messaging house / ICP / buyer persona / JTBD / job / launch tier (T1/T2/T3) / GTM motion / PLG vs SLG / beachhead / crossing the chasm / category design / battlecard / win/loss / trap question / message testing / message pull-through / sales enablement / content adoption / good-better-best / value metric / willingness-to-pay / analyst relations / Magic Quadrant / land and expand / TOFU/MOFU/BOFU / ABM。

### 5.2 外行破绽 (outsider-tells)
- 把 PMM 和 PM (产品管理) 混为一谈 ("PMM 就是帮产品做营销物料的")。
- 把定位等同于 slogan / tagline ("我们的定位就是这句话")。
- 把功能清单当 messaging ("homepage 上多放点产品功能")。
- 把「我们觉得很 punchy」当 messaging 验证。
- 随口说「我们要创造一个新品类」当默认增长动作。
- 把 growth hacking 当定位替代品 ("上几个增长 hack 就行")。
- persona 里写一堆无决策力的人口统计 (年龄/爱好)。

### 5.3 厂商/外行话术拒绝 (the field rejects)
- "We invented a category" (多数是事后包装, 真品类创造极罕见)。
- "Just make it sound more compelling" (没有买家验证的「动听」不是 messaging)。
- "Our competitor is terrible, put it in the battlecard" (FUD 反噬)。
- "Every release needs a big launch" (训练市场无视你)。
- "Growth will fix positioning" (战术不替代战略)。
- "PMM owns the swag and the slides" (接单工具人窄化)。

### 5.4 对话样本库 (industry voice 实战语料)
- "What's the competitive alternative here — including doing nothing?"
- "That's a slogan, not positioning. Who is this for and against what?"
- "Features tell, benefits sell — what job are they hiring us for?"
- "Did we test this with real buyers, or do we just think it's punchy?"
- "What tier is this launch? Not everything needs a tier-1."
- "No FUD in the battlecard — where do we honestly win?"
- "That persona is made up. Where's the win/loss data?"
- "We're not creating a category. Let's win positioning in the one we're in."
- "Who owns this metric, and is the value prop testing with the ICP?"

## 质量基准 + 反模式

### 6.1 什么算「好」 — 5 条可验证基准
1. **定位有真实参照系**: 能说清「客户不买你会用什么替代 (含 status quo)」+ 相对它的独特价值, 不是孤立的形容词堆。
2. **messaging 经买家验证**: 目标买家能复述你的价值 (清晰度/相关性/可信度), 有测试证据不是内部投票。
3. **persona/ICP 有研究支撑**: 来自 closed-won + win/loss + 访谈, 能追溯到证据不是臆测。
4. **launch 分级且量采用**: 有 tier 判断 + 跨职能 plan + 发布后采用度量, 不是一次性庆祝。
5. **竞争物料销售真的用**: battlecard 诚实、按赢的场景组织、有人用且保持更新 (不是过时死文档)。

### 6.2 反模式 — 11 条必避免 (营销/职业伦理 anchor, 不软化)
1. **功能堆砌当定位/messaging**: 列 spec ≠ 定位 ≠ 价值主张; 卖结果不卖 feature 清单。
2. **品类创造当默认动作**: 真品类创造稀有昂贵、多数失败, 轻率建议初创造品类是把罕见打法当模板。
3. **会议室 wordsmith 不测**: 打磨到团队满意 ≠ 市场 land, 不用真实买家测的 messaging 只是自嗨。
4. **PMM 当接单工具人**: 把 PMM 降为「做 deck/图/swag 的」是组织失败信号, 浪费战略杠杆。
5. **功能轰炸式发布**: 每个功能都 T1, 训练市场和内部忽略你的发布。
6. **persona 拍脑袋**: 无研究支撑的 persona 比没有更糟 (给错误决策虚假信心)。
7. **FUD 战卡贬竞品**: 抹黑对手反噬, 诚实差异化 + 承认强项更有说服力。
8. **分析师位当客观背书卖**: Gartner MQ/Forrester Wave 有 pay-to-play 感知 (付费订阅+inquiry+评估), 诚实区分方法论价值 vs 营销背书, 评估投入产出。
9. **PMM ≟ PM 混淆**: PM 造产品/拥有 roadmap, PMM 拥有市场/把产品翻成市场价值——不同职能。
10. **growth hacking 冒充定位**: 增长是渠道/漏斗战术, 定位/品类是战略; 没有定位的增长是往漏桶倒水。
11. **机构自报数字当客观事实**: ODI「约 86% 成功率」(业内估/自陈口径)、PLG「$1B+」、PMA「最快增长岗位」等是自陈口径, 引用当 surrogate 看待不当独立验证。

## 智识谱系

PMM 不是单一学派, 是几条相互借用又分歧的流派交织:

1. **经典定位** (Al Ries & Jack Trout《Positioning》1981 →) → 当代 **April Dunford** 现代化 (把「占领心智」落成 B2B 可执行 5 要素)。核心分歧:经典派「认知 > 现实」vs Dunford 可执行流程化、销售对话驱动。
2. **技术采用-鸿沟** (Geoffrey Moore《Crossing the Chasm》1991)——独立流派, beachhead + whole product + 保龄球道; 当代延伸到 AI/agentic 采用。
3. **JTBD (三分支, 内部分歧)**: Clayton Christensen (HBR/milkshake 奠基) / Bob Moesta (需求侧、定性 switch 访谈、Re-Wired) / Tony Ulwick (ODI、量化 outcome、Strategyn)。分歧:定性叙事 vs 量化测量。
4. **品类设计** (Christopher Lochhead《Play Bigger》/ Category Pirates; Sangram Vajre 的 category creation 倾向)——设计并主导新品类「不同 > 更好」; 自带「多数是事后包装」的诚实警示。
5. **现代 SaaS PMM craft** (Martina Lauchengco《LOVED》/SVPG; Richard King/PMA 职业化; Jason Oakley; Tamara Grominsky)——把 PMM 立为战略职能、4 职责框架、社区/认证基础设施。
6. **messaging-叙事 craft** (Andy Raskin 战略叙事「换世界观」/ Emma Stratton clarity / FletchPMM homepage 定位)——把定位翻成能 land 的语言与叙事。
7. **定价-货币化** (Madhavan Ramanujam《Monetizing Innovation》/Simon-Kucher; Patrick Campbell/ProfitWell; Tamara Grominsky)——围绕价格设计产品、价值定价、打包。
8. **竞争赋能-win/loss** (Peep Laja message/竞争; Jason Oakley 竞争赋能; Klue/Crayon/Clozd 生态)——情报驱动、帮销售赢真实 deal。
9. **增长-PLG/GTM 交界 (边界流派)** (Kyle Poyar PLG 基准 / Wes Bush PLG / Sangram Vajre GTM / Jason Lemkin SaaS GTM; 中国邻域 桑文锋/张溪梦/范冰)——GTM motion 与定价打包与 PMM 重叠, 但增长是战术学科, 边界要划清。
- **媒体枢纽 (跨流派)**: Lenny Rachitsky——本行多数 figures 长访谈的传播节点, 价值在放大与连接而非原创框架。

## 诚实边界

1. **信息截止 2026-05-20**: 工具栈 (Section 3.3 AI/竞争情报) 与「近期变化」衰减最快 (3-6 月), 心智模型 (Section 1) 与谱系 (Section 7) 年更慢。
2. **不能替代真实 GTM 经验**: 本 OS 给的是这行的思维镜片与决策启发式, 不替代在具体公司/品类里跑过定位与 launch 的实战手感。
3. **公开信息有偏差**: PMM thought leadership 有强烈的「成功案例发声、失败沉默」偏差; 大量自报数字 (ODI 86%、PLG $1B、PMA「最快增长岗位」) 是机构自陈, 已标边界, 不当独立验证。
4. **locale 偏英文 (结构性)**: PMM 学科 ~90%+ 一手在英文圈 (canon 全英文、figures 全英文、社区 PMA/Sharebird 英文为主); **中国没有对标的纯 PMM thought leader 群体**, 相关一手落在增长/数据驱动/To-B SaaS 邻域 (神策/GrowingIO/牛透社), 且大量散在黑名单源 (知乎/公众号/woshipm)。任何市场的资深 PMM 仍以英文 figures 为参照系; 本 skill 的 zh-CN 维度刻意标弱。
5. **一手率口径**: manifest 91.5% first-hand (公开 manifest 工具实测), 但其中绝大多数是 surrogate_primary——PMM 一手几乎全在 .com/.io/.co/.substack.com 裸域 (source_verifier 默认判 secondary), 经人工核为作者/厂商/协会一手后标 surrogate "vendor docs"; 这是域名规则的结构性产物, 非数据薄。Gartner/Forrester/G2 主域被列黑名单 (含大量 UGC/PPC), 故分析师/评测站作行业知识保留但无 citable 一手 URL。
6. **B2B 偏向**: 本 OS 重心是 B2B SaaS PMM (销售赋能/竞争情报/分析师关系/win-loss/ABM 多为 B2B 特有); 消费/B2C PMM (品牌叙事/发布为主, 无销售赋能) 与开发者/PLG PMM 的侧重显著不同, 套用前先判 motion。
