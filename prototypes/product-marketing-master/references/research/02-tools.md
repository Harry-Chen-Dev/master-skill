## 1. 必备工具 (Core stack — most PMMs touch these)

These are the categories a generalist PMM uses no matter the company. Ranked by how universally they show up.

### a. Competitive intelligence + battlecards — **the #1 PMM-owned tool category**

- **Tools:** Klue (T02-S001), Crayon (T02-S003), Kompyte/Semrush (T02-S005).
- **场景:** Monitor competitor moves (pricing/launches/messaging), produce **battlecards** sales actually uses, push intel into Slack/Salesforce/CRM at the deal moment.
- **相对优劣:**
  - **Klue** — broadest: CI **+ win-loss in one platform**, 250k+ users (厂商), launched the **Compete Agent** AI (Jun 2025) that auto-builds competitor profiles and answers compete questions in Slack/inbox (T02-S002). Best when you want one vendor for both compete and win-loss.
  - **Crayon** — strongest at automated web monitoring + AI-summarized news → instant battlecard content; deep Salesforce/Slack/Highspot integrations; publishes the **State of CI** benchmark (T02-S004).
  - **Kompyte** — leaner, automation-first, attractive bundled discount if you already pay for **Semrush** (acquired 2022).
- **选型决策点:** Do you need win-loss in the same tool (→ Klue) or just monitoring + battlecards (→ Crayon)? Already on Semrush? (→ Kompyte). Adoption is the real metric — Crayon cites case-study **40% battlecard adoption lift** (厂商); a battlecard nobody opens is shelfware.

### b. Sales enablement (where battlecards/decks/one-pagers live + get measured)

- **Tools:** Highspot (T02-S018), Seismic (T02-S019), Showpad (T02-S020), Mindtickle (T02-S021).
- **场景:** Single source of truth for sales content; governance/versioning; usage analytics (which asset influenced which deal); training/certification.
- **相对优劣:** **Highspot** & **Seismic** are the two enterprise leaders (content management + SmartPages/AI search + analytics; Seismic's **Aura** copilot). **Showpad** leans buyer-engagement + readiness (merged with **Bigtincan** under Vector Capital, late 2025 — watch roadmap churn). **Mindtickle** is readiness/coaching-first (rep ramp, certification).
- **选型决策点:** Pure content + analytics → Highspot/Seismic. Heavy onboarding/readiness need → Mindtickle. PMM's job here is **content quality + governance**, not buying the most seats.

### c. Product/usage analytics (adoption story + PLG signals)

- **Tools:** Pendo (T02-S022), Amplitude (T02-S023), Mixpanel (T02-S024), Heap (T02-S025).
- **场景:** Prove feature adoption post-launch, find activation/retention drivers, segment by persona, run in-app announcements.
- **相对优劣 (官方/第三方公开 positioning):** **Pendo** uniquely pairs analytics **with no-code in-app guides** — PMM can ship a feature announcement without eng, the only one here that does both. **Amplitude** = deepest behavioral cohorts/experimentation (steeper learning curve). **Mixpanel** = most marketer-friendly + transparent pricing. **Heap** = autocapture (no upfront instrumentation; "no data regret" but needs governance to interpret).
- **选型决策点:** Need in-app guides too? → Pendo. Heavy analytical depth/experimentation? → Amplitude. Self-serve for non-analysts? → Mixpanel. PMM rarely *owns* this tool — usually shares it with PM/growth.

### d. Launch / cross-functional coordination

- **Tools:** Notion (T02-S028), Asana (T02-S029), monday.com (T02-S030), Airtable (T02-S031); PM-side: Productboard (T02-S026), Aha! (T02-S027).
- **场景:** Run the **launch tier checklist**, GTM project plan, messaging library, content calendar; ingest customer feedback (Productboard/Aha!) to inform positioning.
- **相对优劣:** **Notion** = docs + lightweight DB (launch hub, messaging doc, FAQ). **Asana** = structured task ownership (who/what/when). **monday** = cross-dept boards/automation. **Airtable** = database-first launch tracker / content calendar. **Productboard/Aha!** are PM tools PMM reads from (feedback → roadmap → "what's launching") rather than owns.
- **选型决策点:** This is **whatever the company already standardized on** — do NOT buy a separate PMM-only PM tool. PMM adapts to the existing system of record.

## 2. 场景特化 (Specialized — by sub-direction)

### Competitive intelligence (deep)
Beyond the core trio: pair CI with **conversation intelligence** (below) — Crayon's benchmark notes teams using conversation-intel tools for compete report **+82% sales effectiveness** (厂商调查, T02-S004).

### Message testing / buyer research
- **Wynter (T02-S008/009)** — the PMM-native one: tests messaging/positioning copy against your **exact ICP** (70k+ verified B2B panel) and returns results **<48h** (厂商). Built by Peep Laja specifically for B2B messaging. Use for "does this homepage/positioning land?"
- **UserTesting (T02-S010)** — video-based concept/UX/message testing, large diverse audience (Forrester Wave leader, 第三方).
- **Maze (T02-S011)** — fast unmoderated tests/surveys, research-grade AI.
- **Sprig (T02-S012)** — in-product micro-surveys + AI analysis (catch users in-context).
- **dscout (T02-S013)** — mobile/diary qualitative depth (Diary/Live missions).
- **Great Question (T02-S014) + User Interviews (T02-S015)** — research **ops + recruiting**; GQ runs the whole study, UI supplies a **6M+ participant panel** (now integrated into GQ).
- **选型决策点:** B2B *messaging/positioning* validation → **Wynter**. Need to *recruit + run* qual studies → Great Question + User Interviews. In-product behavioral feedback → Sprig. Concept/UX video → UserTesting/Maze.

### Win-loss
- **Clozd (T02-S006)** — software + done-for-you research service; AI-assisted interviews, Decision Drivers, "Ask Clozd" search, Slack/CRM sharing.
- **Klue Win-Loss / DoubleCheck (T02-S007)** — DoubleCheck was acquired by **Klue (Jan 2023)**; win-loss now consolidated under Klue alongside CI.
- **选型决策点:** Want win-loss in the same platform as CI → Klue. Want a dedicated specialist (incl. service-led interviews) → Clozd.

### Conversation intelligence (used by PMM as a *research* source, not owned)
- **Gong (T02-S016)** — revenue/conversation intel; PMM mines real calls for objection language, which value props resonate per persona, competitor mentions (case study: **+32% buyer response** by matching value props to personas, 厂商).
- **Chorus / ZoomInfo (T02-S017)** — same use; Chorus.ai acquired by ZoomInfo (2021).
- **选型决策点:** Usually already owned by RevOps/Sales — PMM should get read access rather than buy it. It's the cheapest "voice of customer" you already have.

### Pricing
- **ProfitWell Metrics / Paddle (T02-S037)** — free subscription metrics (MRR, churn, ARPU, LTV) benchmarked vs 30k+ SaaS (厂商); the *data* layer. ProfitWell is now part of **Paddle**.
- **Simon-Kucher (T02-S038)** — the *methodology* layer: pricing/monetization **consultancy** (founded 1985, "world's leading pricing specialist"). Not software — engaged for pricing strategy/willingness-to-pay studies.
- **选型决策点:** Need ongoing subscription metrics → ProfitWell. Need a one-off pricing-strategy overhaul / WTP research → Simon-Kucher (or run your own van-Westendorp via Wynter).

### Review-site operation (PMM operates *on* these — they are subjects, not internal tools)
- **G2**, **TrustRadius**, **Capterra**, **PeerSpot (T02-S036)**. （注: G2 / TrustRadius / Capterra 的主域被 source_verifier 列入黑名单——这类评测站含大量 UGC + PPC 排序, 不作 citable 一手源; 它们仍是 PMM 的运营对象, 这里作行业知识保留。G2 反激励政策见 legal.g2.com community-guidelines T02-S033。）
- **场景:** PMM runs review-gen campaigns, manages category presence, mines reviews for competitor strengths/weaknesses (Klue can auto-mine these, T02-S044), buys buyer-intent data.
- **相对优劣:** **G2** = largest (~6M reviews; **acquired Capterra/Software Advice/GetApp from Gartner, Feb 2026** — now one network). **TrustRadius** = long-form depth, smaller volume. **Capterra** = SMB/PPC-driven. **PeerSpot** = enterprise IT/security/DevOps niche.
- **选型决策点:** B2B SaaS broad → G2 first. Enterprise infra/security buyer → PeerSpot. Budget for buyer-intent → G2/TrustRadius paid tiers (TrustRadius Customer Voice ≈ $30k/product/yr, 第三方).

### Content / landing pages
- **Webflow (T02-S039/040)** — visual website + CMS so **marketers self-serve landing pages** without eng (200k+ companies, 厂商). Use for launch/campaign pages, programmatic CMS pages.
- **选型决策点:** Need marketer autonomy + design control without a dev queue → Webflow. If on an existing CMS (HubSpot/WordPress/headless), don't add Webflow just for PMM.

## 3. 新兴 — AI (近 12 月)

AI moved from novelty to default in the compete/messaging workflow. Crayon's benchmark: compete teams saw **+76% YoY AI adoption**, **60% now use AI daily** (+25% YoY); top uses = summarizing content + analyzing large data (厂商调查, T02-S004).

- **General LLMs — ChatGPT / Claude:** drafting messaging variants, summarizing competitor sites/earnings, first-pass FAQ/objection handling, synthesizing call/research transcripts. **Caveat:** generic LLMs are reactive, lack your domain context, and can hallucinate competitor facts — Klue explicitly positions Compete Agent against "ChatGPT/internal LLMs" for this reason (T02-S002). Use for *drafts*, never as a fact source on competitors.
- **Embedded compete AI — Klue Compete Agent (T02-S002, Jun 2025)** & **Crayon AI / Klue AI (T02-S003/044):** purpose-built, trained on *your* competitors/positioning, human-in-the-loop verification, delivers deal-specific intel in Slack/inbox. Klue also auto-generates competitor **strengths/weaknesses from review sites** (T02-S044).
- **Conversation AI — Gong AI agents (T02-S016):** AI Trackers surface which value props resonate per persona from real calls.
- **Marketing copy AI — Jasper (T02-S041), Anyword (T02-S042), Writer (T02-S043):** Jasper = full-stack marketing copy + brand voice + agents; Anyword = **Predictive Performance Score** (forecasts copy performance pre-publish); Writer = enterprise, **brand-trained custom LLMs** + governance.
- **选型决策点:** Compete intel → embedded compete AI (verified) over raw ChatGPT. Brand-governed enterprise copy at scale → Writer. Performance-predicted ad/copy → Anyword. Versatile drafting → Jasper or general LLM. **Always keep human verification on any competitor-facing AI output.**

## 4. 避坑清单 (Anti-patterns — selection decision tree)

**These are the mistakes outsiders make. Stated bluntly, not softened.**

1. **工具堆砌 ≠ 流程 (a stack is not a system).** Buying Klue + Gong + Pendo + Highspot + Wynter does not make you a PMM org. Tools amplify an existing process; with no positioning, no launch tiers, and no owner, you've just bought expensive shelfware. **Decision rule:** name the workflow and the human owner *before* the purchase. If nobody opens it weekly, cancel it.

2. **Battlecards stuffed with FUD backfire.** Loading a battlecard with fear/uncertainty/doubt and a comparison table "entirely stacked in your favor" destroys rep credibility — buyers and reps both see through it. Klue's own counter-FUD guidance and Crayon's role-based-battlecard guidance both say: **acknowledge competitor strengths, then pivot to where you genuinely win** (T02-S049, T02-S050). A battlecard that pretends competitors have no strengths is the one reps stop trusting.

3. **Persona/positioning tools filled by gut, not research.** Productboard, a Notion persona template, or a positioning canvas are empty containers. Filling them from internal opinion ("we think the buyer cares about X") instead of real buyer input is the classic outsider error. **Decision rule:** every persona/positioning field traces to evidence — Wynter message tests, Gong/Chorus call transcripts, Clozd win-loss interviews, or Great Question studies. The tool doesn't do the research; it stores it.

4. **Gaming review sites violates platform rules and backfires.** Soliciting only positive reviews, conditioning incentives on positivity, or planting fake reviews **breaches G2's Community Guidelines** (and FTC rules) — G2 prohibits inauthentic reviews, never conditions incentives on a positive opinion, labels incentivized reviews, and validates reviewer identity (T02-S033). Caught manipulation = removed reviews + reputational damage. **Decision rule:** run honest, broad review-gen (ask all customers, accept the negatives) — that's also what produces credible category presence.

5. **Buying conversation intelligence PMM doesn't need to own.** Gong/Chorus are usually already paid for by Sales/RevOps. A PMM purchasing a *separate* license is wasted budget — **ask for read access first.**

6. **Adding a redundant PM/launch tool.** Don't impose a PMM-only Asana/monday/Notion instance parallel to the company's system of record. Adapt to what exists; fragmenting the launch plan across two tools loses cross-functional visibility — the entire point of the tool.

### Quick selection decision tree

- *Need competitor monitoring + battlecards?* → Crayon (monitoring) / Klue (CI+win-loss in one) / Kompyte (if on Semrush).
- *Need to validate messaging/positioning?* → Wynter (B2B ICP, fast). Don't guess.
- *Need to know why deals are won/lost?* → Clozd or Klue Win-Loss. Don't infer from CRM stages.
- *Need adoption proof + in-app nudges?* → Pendo. Pure depth → Amplitude.
- *Need to host sales content + measure it?* → Highspot/Seismic. Don't email decks.
- *Need launch coordination?* → use the company's existing PM tool. Don't buy a new one.
- *Reaching for AI on competitors?* → embedded compete AI (verified), not raw ChatGPT.
