> 调研日期：2026-06-03。本 track 覆盖：广义运营**工具栈**（合规清单 / 课程设计框架 / 安全保险工具 / 运营 SaaS / 获客营销 / 场景特化器材与平台）+ 选型决策树 + 避坑清单。

## 诚实边界（先读）

本行业的「工具」**不是软件为主，而是广义运营工具栈**。撒网后的核心判断：

1. **「工具栈成熟度」结论：缺成熟的全行业垂直 SaaS，主力靠通用工具拼装 + 标准/框架当 checklist。** 一线机构（尤其 C 端营地 / 中小研学机构）的实际工具栈 = 「微信生态（小程序 + 视频号）+ 通用表单（金数据 / 腾讯文档收集表）+ 通用知识付费/分销（小鹅通 / 有赞教育）+ OTA/本地生活分销（抖音 / 美团 / 携程）」拼装，再叠加「标准与政策清单 + 课程设计框架 + 三层险栈」这类**非软件工具**。垂直 SaaS（旅科云、好多研学、朗睿等）真实存在，但**主要面向 B 端政府/教育局/基地侧的「一网通办」监管—备案场景**，C 端家庭付费机构渗透率低、产品认知度低、行业自评「炒概念、实用性弱、售后难保障」(evidence: [T02-S012])。

2. **本行业「最重的工具」是合规与方法，不是 App。** 真正决定一个研学/营地产品成败的「工具」是：①能不能对标 LB/T 054-2025 / T/CATS / ACA 做合规自检；②课程有没有用 Kolb 圈 / PBL / 5E / UbD 这套设计框架；③安全保险三层栈和师生比/营辅比配齐没有。软件只是履约层。故本 track 把**标准清单 + 课程框架 + 安全保险工具**作为「必备层」主体，SaaS 反而多落在「场景特化」。

3. **信源结构性偏低说明（同 Wave 1）**：zh-CN 一手运营内容大量沉淀在已被黑名单排除的微信公众号/知乎，故运营/获客层一手率偏低，大量启用 surrogate_primary（标准原文 / vendor 官方产品页 / 协会官档 / 律所合规长稿 / 政府平台）。**SaaS 产品名已逐一 web 核实真实存在**（vendor 官方域名可访问 + 第三方/政府案例旁证），未编造产品名；找不到成熟垂直产品的环节（如 C 端一体化）已诚实写「拼装」。

> bucket 规则（本 track 适用）：`.gov.cn` / `.edu` / 国家标准平台 / 政府平台 = verified_primary；**vendor 官方产品页/官档 = verified_primary（产品「真实存在 + 功能」的一手，但 vendor 自家营销话术不作市占/效果背书）**；协会/标准组织官网一手（ACA/LNT/PBLWorks/CCEA）= surrogate_primary（note 含「协会/标准组织官档」）；律所合规长稿 / 行业媒体 = secondary。`source_verifier.py classify` 对 vendor/协会域名默认给 secondary，已按本规则人工升级并在 note 标注依据。zh-CN 黑名单（知乎/微信/百科/CSDN/36kr/网易号）一律不入 Manifest。

## 必备层工具卡片

### 1. LB/T 054-2025《研学旅游服务要求》（+ 091 出境 / 092 课程线路）

- **One-liner**: 研学服务的国家级行业标准，从业者把它当「资质达标 + 选基地 + 设计营 + 合同条款」的硬 checklist；2025-05-19 起替代旧版 054-2016。 (evidence: [T02-S001])
- **Tier**: 必备（合规/政策工具，非软件）
- **Maintainer / Owner**: 文化和旅游部（2018 机构改革后从国家旅游局接手）— https://zwgk.mct.gov.cn
- **License**: 公开行业标准（推荐性，LB/T）
- **Maturity signals**: 发布 2025-02-19、实施 2025-05-19（last checked 2026-06-03）；同批拆出 LB/T 091-2025《出境研学旅游服务要求》、LB/T 092-2025《研学旅游课程与线路设计指南》。 (evidence: [T02-S001])
- **典型使用场景**:
  1. 投标/承接学校研学项目前，逐条自检人员配置（研学导师+导游+安全员+项目组长）、安全管理、投诉处理是否达标。
  2. 设计出境营时，对照 LB/T 091-2025 补境外特有合规项。
  3. 设计课程/线路时，把 LB/T 092-2025 当课程与线路设计的官方框架。
- **不适合 / 替代方案**: 标准是推荐性非强制，**不能替代地方强制规范**——北京等地有更严的地标（见 S029），承接当地业务以地标为准。

## 选型决策树

> 节点数 8（A1/A2 通用拼装、B1/B2/B3 运营 SaaS、C1/C2 场景器材、D 合规/安全前置门）。每节点可追溯到工具卡片证据。

### 决策树 A：你的核心需求是「报名+收款+轻运营」（中小机构 / 单一夏令营起步）

#### Branch A1: 极轻、预算有限、营期少
→ **推荐**: 微信小程序 + 金数据/腾讯文档收集表（报名+健康信息+监护授权）+ 微信支付。理由：零/低成本、上手快、行业默认拼装解 (evidence: [T02-S019, T02-S021])。
→ **不推荐**: 一上来买垂直研学 SaaS（旅科云等）——营期少时偏重、价格不透明、ROI 低 (evidence: [T02-S010])。
→ 真实案例：行业普遍做法，因垂直 SaaS「选择少、实用性弱」中小机构多用通用工具拼装 (evidence: [T02-S012])。

#### Branch A2: 需要线上课/分销裂变招生
→ **推荐**: 叠加 **小鹅通**（行前线上课+训练营+内容分销裂变）或 **有赞教育**（交易+课程分销市场，含游学类目） (evidence: [T02-S015, T02-S017])。二选一原则：偏「内容/私域/裂变」选小鹅通；偏「交易/微商城/分销佣金」选有赞。
→ **不推荐**: 只靠小鹅通做线下履约——它无营期排期/营员档案 (evidence: [T02-S016])。

### 决策树 B：你已有规模、需要成体系运营（多营期 / 多渠道 / 安全档案）

#### Branch B1: 机构侧成体系运营（基地/中大型机构）
→ **推荐**: 垂直研学 SaaS（**旅科云**：课程+安全监管+行程+评估+合规一体） (evidence: [T02-S010])。差异点是「安全监管（位置追踪+电子档案）+ 合规模块」，通用工具补不齐。
→ **替代**: 朗睿（若要「自营+商家入驻」平台化） (evidence: [T02-S011])。
→ caveat：垂直 SaaS 市占/效果 vendor 未披露，选型前要试用+索取真实客户案例核验。

#### Branch B2: 已机构化、有固定班型与续费（周末班+营混合）
→ **推荐**: **校管家**（招生/教务/财务/家校/续费八模块）做教务财务底座 (evidence: [T02-S014])，营期产品交易侧叠加有赞/小鹅通。
→ **不推荐**: 用校管家硬套纯营期制夏令营——它为学科/兴趣班教务设计，无营期/线路/安全模块 (evidence: [T02-S014])。

#### Branch B3: 你是教育局/区域监管方（不是卖营的机构）
→ **推荐**: 「研学一网通办」政府平台（好多研学 / 区域定制如东营黄河云平台）做备案—审批—监督—报告 5 端 (evidence: [T02-S013, T02-S034])。

## 避坑清单（≥ 5，实为 9 条）

- ❌ **不要把 OTA / 本地生活平台（携程/美团/抖音团购）当成「研学课程工具」**：它们是分销/获客渠道，重旅游轻教育；用它们的旅游逻辑做产品 = 「只游不学」，2024-11 文旅部专门发文治理伪研学 (evidence: [T02-S030])。渠道 ≠ 课程设计工具。
- ❌ **不要无安全/保险工具裸奔上线**：只买一份意外险就出团是最常见致命错——必须三层险栈（校方责任险+旅行社责任险+人身意外险）+ 师生比/营辅比达标 + 应急预案，否则出事即重大法律/舆情风险 (evidence: [T02-S009])。
- ❌ **不要把 T/CATS 研学指导师证当「国家职业资格准入证」宣传**：它是团体标准+测评，非国家职业资格；夸大宣传是行业普遍合规破绽 (evidence: [T02-S004])。
- ❌ **不要把通用教培系统（校管家）硬套纯营期制夏令营**：校管家为学科/兴趣班教务（排课/续费）设计，无营期/线路/安全合规模块，错位使用管理成本反而更高 (evidence: [T02-S014])。
- ❌ **不要把 AI 研学方案生成器的输出直接当成品**：AI 易堆砌景点、缺反思与评价（「只游不学」），必须当草稿、再用 Kolb/PBL/UbD 人工把关教育性与安全 (evidence: [T02-S025])。
- ❌ **不要迷信「智慧研学/营地 SaaS」营销话术就盲目采购**：行业自评垂直 SaaS「炒概念、收费不合理、实用性小、售后难保障」；采购前务必试用+索取真实客户案例，vendor 自报的市占/装机量需独立核验 (evidence: [T02-S012])。
- ❌ **不要把营地「营辅比」当成学校「师生比」套用**：营地按年龄分级远严于学校（ACA 住宿营低龄 1:6），套用班级 1:40 是安全红线违规 (evidence: [T02-S005])。
- ❌ **不要继续引用已废止的 LB/T 054-2016**：2025-05-19 起 054-2025《研学旅游服务要求》已替代，合同/手册/培训仍写旧标号=合规知识过期破绽 (evidence: [T02-S001])。
- ❌ **不要把「监管侧一网通办平台」当机构经营系统**：好多研学/区域云平台解决教育局备案审批监督，不含机构自己的招生/收款/分销，机构侧仍要另配运营栈 (evidence: [T02-S013])。

## Phase 2 提炼提示

### 反复出现 ≥ 3 处都强调的「工具选型原则」（候选 playbook 规则）

- **原则 1：合规与安全工具是前置门，先于一切软件选型。**（出现于 LB/T 054 / 三层险栈 / 师生比配比 / 应急预案 / 文旅部 2024-11 通知）——判断式：「师生比/应急预案/三层险栈/对标 LB/T 054-2025 任一不达标，则无论用什么 SaaS 都不准成行。」 (evidence: [T02-S001, T02-S009, T02-S005, T02-S030])
- **原则 2：本行业缺成熟全行业垂直 SaaS，默认解是「通用工具拼装」，规模化后才上垂直/专用系统。**（出现于 营地教育网行业自评 / 通用拼装底座普遍采用 / 垂直 SaaS 价格不透明且 B 端监管侧为主）——判断式：「营期少→小程序+金数据+小鹅通/有赞拼装；成体系/有安全档案需求→才上垂直研学 SaaS。」 (evidence: [T02-S012, T02-S019, T02-S010])
- **原则 3：分销渠道 ≠ 课程工具，工具按『获客/交易/履约/课程设计/合规』分层，别混用。**（出现于 OTA 误用避坑 / 小鹅通无履约 / 校管家错位 / AI 生成当草稿）——判断式：「抖音美团做获客、有赞小鹅通做交易分销、垂直 SaaS/拼装做履约、Kolb/PBL 做课程、LB/T 做合规——一个工具想全包必有短板。」 (evidence: [T02-S030, T02-S016, T02-S014])
- **原则 4：课程设计工具（框架）比软件更决定产品成败，且必须含「反思+评价」闭环。**（出现于 Kolb/PBL/UbD/5E 必备层 + AI 生成避坑 + OTA 避坑）——判断式：「方案若无行后反思与成果产出/可测评价，则是旅游不是研学，再好的报名系统也救不了。」 (evidence: [T02-S008, T02-S025])

### 显著的工具流派分裂（候选智识谱系条目）

- **「通用工具拼装派」 vs 「垂直 SaaS 一体派」**：
  - 拼装派（代表工具：微信小程序+金数据/腾讯文档+小鹅通/有赞+OTA 分销）——低成本、灵活、行业当下主流，缺点是多营期/多渠道时管理碎片化 (evidence: [T02-S019, T02-S015])。
  - 垂直一体派（代表工具：旅科云/朗睿/好多研学）——课程-安全-行程-合规一体，缺点是价格不透明、B 端监管侧为主、C 端渗透低、行业认知「炒概念」 (evidence: [T02-S010, T02-S012])。
  - 这条分裂本质是 Track 04 canon「标准化课程 vs 在地化资源」「教育属性 vs 旅游属性」在工具层的投影。
- **「教育属性派工具」 vs 「旅游属性派工具」**（与 Track 04/06 的核心矛盾同构）：
  - 教育派（课程设计框架 Kolb/PBL/UbD/5E + 研学手册 + 物种识别/STEAM 器材）强调学习闭环。
  - 旅游派（OTA/本地生活分销 + 通用报名收款）强调流量与交易。
  - 工具选择即立场：偏前者=「研学是课程」，偏后者易滑向「只游不学」。

### 新兴工具信号

- 当前活跃/上升的新工具数：3（研学 AI 方案生成器 yxueai / 中经网络 AI 线路系统 / 视频号-抖音 AI 投流私域打法），全部 `stability: experimental`、`Decay risk: high`。

### 冷僻 / 信号薄弱（供 Phase 2.8 诚实边界）

- 必备层 11 个 ✅（但**主体是「非软件工具」**：标准清单 6 + 课程框架 4 + 安全保险 3，真正的「软件必备」仅通用拼装底座 1 项）——这是本行业工具栈的**结构性特征**，须在 SKILL.md 诚实边界明说「本行业必备工具以合规清单+课程框架+安全工具为主，软件层不成熟」。
