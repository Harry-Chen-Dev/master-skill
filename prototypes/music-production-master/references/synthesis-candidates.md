## 候选 playbook 规则 (target 6-9)

| Pattern | 来源 |
|---------|------|
| 不确定判断好不好 → 响度匹配后对照商业 reference | T03 / T04-S001/S022 |
| 想加 X（轨/EQ/压缩/响度）→ 先问能不能减/不加 | T03 / T04 subtractive |
| 下游难 → 回上游解决，别在下游补救 | T03 / T04 |
| 用了任何 AI/自动工具 → 当起点 + null/A-B + 人工接管 | T02 / T03 |
| 交付/定稿前 → 在尽量多真实回放环境复核（mono/小系统/低音量/多平台） | T03 / T04 |
| 混音糊/挤(mud) → 先减法 EQ + reference，而非加 | T04-S001/S022 |
| 想加厚不压扁瞬态 → 并行压缩（NY compression） | T02-S019 / T04-S017/S018 |
| 母带 → 命中平台 LUFS/true-peak 上限优先保动态，不冲最响 | T03/T04 LUFS |
| 选工具 → 先问"哪一环、什么瓶颈"再选，不追"最好的工具" | T02 决策树 |
| 房间未处理 → 先声学处理+reference，再谈贵箱/插件 | T02-S018 |

## 候选 Agentic Protocol 维度

| 维度 | 推导自哪个 mental model |
