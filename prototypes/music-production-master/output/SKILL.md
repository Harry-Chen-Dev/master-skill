## 心智模型

> 全行第一性张力有两条主轴：(轴 A) serve-the-song / emotion-first ⇄ technical-correctness；(轴 B) mix-by-ears / 直觉 ⇄ mix-by-metrics / 原理。心智模型沿这两轴 + 衍生张力提炼。**注意 tacit-knowledge ceiling**（见 §8）：以下模型给的是框架、流派与决策原则，"黄金耳朵"靠 reference + 千小时积累，无法完全言传。

### 1.1 服务歌优先于服务技术 (serve the song)

- (figures: Rick Rubin / Serban Ghenea / Andrew Scheps)
- **一句话**：技术、音色、个人风格全都服从于"这首歌要表达什么"——好不好的最终裁判是情绪与歌本身，不是参数表（"if it sounds good, it is good"）。
- **应用方式**：面对任何处理决策（要不要加这条轨/这个 EQ/这台压缩/这点响度），先问"它服务于歌的情绪吗，还是只服务于我的手痒/炫技/标志音色？"。混音师不强加自己的 sound（Ghenea "no bag of tricks"），制作人是"reducer"剥到本质（Rubin）。
- **局限**：放大度——这是通用创作道理（"以用户/作品为中心"）在音频 craft 的强放大版；排他性体现在它被具体化为可操作纪律（serve-the-song 优先于 kick/snare 的技术完美、优先于个人签名音色），而非泛泛的"以人为本"。对纯技术交付场景（如严格按广播 R128 -23 LUFS 规范母带）则让位于硬指标。
- **evidence**: [T01-S021, T01-S007, T04-S022]

### 1.2 减法优先 / 每个处理都要能说出理由 (less but knowing why)

- (figures: Andrew Scheps / Rick Rubin / Greg Calbi)
- **一句话**：默认不处理；做得太多而非太少是这一行最常见的病——加任何东西（轨/EQ/压缩/响度）之前，先确认能不能减、不加，且每一次处理都要能讲出"我为什么这么做"。
- **应用方式**：起手用空白（不带 active 处理的）模板（Scheps "Art of Letting Go" 模板默认关闭一切染色），先减问题频段（mud/harsh）清出空间再谈加色；编曲阶段主动删轨而非加轨；母带阶段"有时最好的处理是什么都不做"（Calbi）。判断资深与否的快锚点：他知道"什么时候可以省/不做"。
- **局限**：放大度——"少即是多"是普遍审美原则，在本行被特化为"先减后加 + 每个动作可解释"的可执行纪律，且有边界：character 混音派（Elmhirst/Marroquin）和有意 maximalist 的制作（密集 trap/EDM 编曲）会刻意"做更多"，减法不是无条件律令。
- **evidence**: [T01-S001, T01-S035, T04-S022]

### 1.3 所有听感判断都要外部校准（耳朵、房间、AI 营销都会骗你）

- (figures: Mike Senior / Bob Katz / Chris Lord-Alge)
- **一句话**：人耳会疲劳、房间会撒谎、"更响/更不一样"会被误判成"更好"、AI 营销会兜售安慰剂——所以所有听感判断都要靠外部锚点校准：响度匹配后的 reference track、校准监听、mono/多系统/低音量复核、null test。
- **应用方式**：不确定一个判断好不好时，响度匹配后 A/B 商业 reference（不是凭记忆/凭房间）；混完折 mono 查相位、过手机/车载/耳机、降到对话音量复核平衡（CLA "低音量监听才是真 secret"）；母带在校准监听（≈83 dBSPL，K-System，官方 K-System 协议 T06-S011）上对 reference 母带；验证任何 AI/插件"是否真更好"时跑 null test（Worrall）。
- **局限**：reference 只能代偿、不能替代耳朵训练（tacit ceiling，见 §8）；选错 reference（不同 genre/不同制作意图）会把混音带偏；过度依赖 reference 会抹掉作品个性。仅在"有可对照的成熟商业参照系"时强成立。
- **evidence**: [T01-S052, T01-S037, T04-S001]

### 1.4 动态 > 绝对响度（normalization 时代，响度不再是竞争力）

- (figures: Bob Katz / Bob Ludwig / Greg Calbi)
- **一句话**：流媒体按统一目标（≈ -14 LUFS，平台官方文档 T04-S021）归一回放后，过压只赔动态、不赚响度（超出部分被 normalization 拉回），所以"好声音"与"响"已被解耦——动态范围比绝对响度更有竞争力。
- **应用方式**：母带把响度推到平台 target 即止，留 true-peak ≤ -1 dBTP 余量（业内/官方共识，T03-S004）；遇到"做更响=更专业"的冲动时，提醒自己 -14 LUFS 是上限触发点不是必须命中的硬目标（动态母带可更低）；批评/识别"响度战争遗毒"的过压成品。
- **局限**：放大度——这是 normalization 这一具体制度催生的、强排他于音频后期的判断（在没有响度归一的媒介如某些俱乐部播放/黑胶刻片，博弈不同）；且"响度战争是否真结束"本身仍是开放争议（Spotify -14 下仍有"迷你响度战"），不可武断。平台 target 无公告可调（Decay risk: medium）。
- **evidence**: [T01-S046, T04-S008, T04-S021]

### 1.5 AI / 自动输出是起点不是终点（近 12 个月最活跃的判断）

- (figures: Bobby Owsinski / Dan Worrall / Warren Huart)
- **一句话**：AI EQ、AI 母带、stem 分离、mix assistant 在近 12 个月从"边缘玩具"变"主流插件内置功能"，但资深人一律把 AI 输出当**起点**——必须响度匹配 A/B + null test + 人工用耳朵接管，且有原始多轨时根本不用 stem 分离（分离永远有伪影）。
- **应用方式**：用了任何 AI/自动工具（Ozone Master Assistant / smart:EQ / LANDR / Logic Stem Splitter），先问"这是起点还是我打算直接发的终点？"；用 null/A-B（响度匹配）戳穿"变化=改进"的错觉；AI 拆出的 stem 先做伪影 QC，且通常不让它进母带链。
- **局限**：这是一个把通用"AI 是工具不是替代"原则放大到音频 craft 的强时效模型（Decay risk: high，AI 模型月级迭代，6 个月后最佳引擎/最佳实践可能翻新）；对真正只要"够用、快、便宜"的 demo/独立快发行，一键 AI 母带的成本-收益权衡与商业发行不同，不是一刀切禁用。
- **evidence**: [T02-S010, T03-S020, T01-S060]

## 标准 Playbook

> 形如 `**如果 X**，则 Y`。Phase 3 `cli_writer.py` 会按主题聚类成决策树脚本。

1. **如果不确定一个听感判断好不好（混音/母带某步是否到位）**，则**先把自己的版本和一首同 genre 的商业 reference 做响度匹配（gain-match），再 A/B 对照**，而不是凭记忆、凭房间、凭"听起来不一样"下结论——更响的总显得更好，没匹配响度的 A/B 等于自我欺骗。案例：Senior 把 reference A/B 列为对抗房间谎言与耳疲劳的第一工具；混音师在 mud/harsh 诊断上都靠它（T04-S001 / T04-S022）。

2. **如果你想加任何东西（多一条轨、一段 EQ boost、一台压缩、再推一点响度）**，则**先问"能不能改成减、或干脆不加"**，默认不处理、每个处理都要能说出理由。案例：Scheps 模板起手关闭一切染色处理；Calbi"有时最好的处理是什么都不做"；编曲阶段资深人主动删轨（T01-S035 / T03-S007）。

3. **如果下游阶段很难搞（混音怎么都不清晰、母带救不回）**，则**回上游解决而不是在下游补救**——不要 "we'll fix it in the mix / in the master"。案例：编曲就给人声留空间、别让 5 件乐器挤 200-500Hz；tracking 阶段"录好源头=混音赢一半"，先调话筒/摆位/房间而非堆插件（T03-S001 / T02-S018）。

4. **如果你用了任何 AI / 自动工具（AI EQ、AI 母带、mix assistant、stem 分离）**，则**把它当起点不当终点：响度匹配 A/B + null test 验证"真更好"而非"只是变了" + 人工用耳朵接管**；且有原始多轨时根本别用 stem 分离。案例：Ozone Master Assistant / LANDR 输出被业内批偏 harsh/digital，必须 A/B 原始；Logic 本地 stem 拆分也要先做伪影 QC（T02-S010 / T03-S020）。

5. **如果准备定稿/交付（混音 print 前、母带交付前）**，则**在尽量多的真实回放环境复核**：折 mono 查相位、过手机/车载/耳机、降到对话音量校平衡、母带预演多平台 normalization、多格式（含 Atmos 时附立体声兜底）。案例：CLA"低音量监听是真 secret"；混音 mono 兼容是相位检查的硬关卡（T03-S001 / T01-S037）。

6. **如果混音听起来"糊（mud）/挤/刺耳（harsh）"**，则**先做减法 EQ（减 200-500Hz mud / 2-5kHz harsh）清出空间，再考虑加色**，必要时用动态 EQ/soothe 压共振；不要一不够就 boost。案例：subtractive EQ 是 canon 跨多源共识的混音第一性手法（T04-S001 / T02-S001）。

7. **如果想让元素"更厚、更有冲击力又不想压扁瞬态"**，则**用并行压缩（NY compression）：干信号保瞬态 + 一路重压副本垫底并行混合**，而不是把单链压得更狠。案例：起于 1970-80s 纽约棚 SSL/1176 重压劈分；Brauer 把它发展成 multing 多总线分压、Scheps 发展成 3-bus（T04-S018 / T04-S017）。

8. **如果在做母带的响度/限制**，则**命中平台 LUFS target（≈ -14 streaming）即止、留 true-peak ≤ -1 dBTP、优先保动态，并只在最后一次降位深时加一次 dither**——-14 是上限触发点不是必须命中的硬目标，过压换不来响度只赔动态。案例：normalization 机制下过压被回拉（T03-S006 / T04-S021）。

9. **如果在选 DAW / 插件 / 硬件**，则**先定位"在 pipeline 哪一环、当前瓶颈是什么"再选工具，而不是追"最好的工具"**；房间未处理时先做声学处理 + reference，别先信贵监听/贵插件。案例：多 DAW 工作流已是常态（编曲 Ableton → 录混 Pro Tools → 母带 Ozone）；NS-10 再好房间不处理也只是听房间的谎言（T02-S003 / T02-S018）。

## 工具栈与选型决策树

> 直接消化 Track 02 的三层结构。一致性 sanity-check：必备层 9 个（≥3 ✅）、场景特化 8 个、新兴 5 个、避坑 ≥ 5 条（实际 7 条）、决策树 4 大分支多节点且每节点有证据 ✅。插件采用率无官方 survey（不像 DAW 有 Production Expert survey），必备层插件"≥80%"是基于 canon 反复点名 + 多评测共识的合理推断（业内估计），已标 caveat。

### 3.1 三层工具栈

**必备层（行业地基，≥ 80% 业内估计 / canon 反复点名）**：
- **Pro Tools** — 录音棚/post 录音剪辑事实标准、session 交换通用语（pro 棚 37.2%、post 63.8%，T02-S003）。
- **FabFilter Pro-Q** — 现代 ITB 默认精密 EQ（频谱+动态+M/S）。官方价约 USD 179（T02-S002）。
- **1176 / LA-2A / SSL bus comp（含 UAD/Waves/PA 复刻）** — 三件套压缩原型（FET 快/光学慢/总线胶水），几乎所有混音的默认动态配置（T02-S012, T02-S019）。
- **iZotope Ozone** — ITB 母带事实默认套件 + AI Master Assistant 起点。约 $299–$499（业内常报价，T02-S022）。
- **Valhalla 混响（VintageVerb/Room/Plate）** — 约 $50（官方价）复刻 Lexicon/EMT，被海量 producer 当参考混响（T02-S011）。
- **Auto-Tune / Melodyne** — 修音双标准：实时音染 vs 离线外科透明+复音（T02-S014）。
- **限制器 Pro-L 2 / Ozone Maximizer** — 母带末端响度限制默认（T02-S005）。
- **校准近场监听（Genelec / Yamaha HS / NS-10 传统）+ 声学处理** — 任何混音的"真相参考"，房间+监听优先于插件（T02-S018）。
- **Apollo / RME（接口/转换器）** — 录音棚级 AD/DA + 低延迟，Apollo 带 UAD DSP（T02-S012）。

**场景特化层**：Ableton Live（电子/现场/编曲）、FL Studio（trap/hip-hop beat 母语）、Logic Pro（Mac 性价比 + 本地 Stem Splitter）、oeksound soothe2/3（外科共振抑制）、Serum 2 / Kontakt（音源）、Splice（样本订阅 + Rent-to-Own）、Reaper（轻量可定制极低价）、U47/SM7B/SM57（话筒源头三件套）。

**新兴 / 实验层（近 12 个月，Decay risk: high 为主）**：FabFilter Pro-Q 4（2024-12，加 Spectral Dynamics+Character）、iZotope Ozone 12（2025-09，加 ML Bass Control/Unlimiter+IRC 5）、sonible smart:EQ 4（AI EQ）、AI stem 分离群（Logic Stem Splitter / RipX / Moises / RX Rebalance）、在线 AI 母带（LANDR / CloudBounce / eMastered）。

### 3.3 避坑清单（≥ 5）

- ❌ 不要为"更响"在母带过压——normalization 时代过压只换动态损失不换响度（T02-S010）。
- ❌ 不要把一键 AI 母带/Assistant 输出当成品直接发——是起点不是终点，偏 harsh/digital，必须 A/B（T02-S006）。
- ❌ 不要在没处理的房间里相信贵监听——先声学处理 + reference track（T02-S018）。
- ❌ 不要迷信"模拟 summing/硬件 warmth"忽略盲测——null test 显示干净 summing 与 ITB level-match 后基本抵消，mojo 多来自饱和/染色（T02-S020）。
- ❌ 不要用 AI stem 分离替代原始多轨——有多轨永远用多轨，分离有伪影进母带链会放大（T02-S009）。
- ❌ 不要把 Auto-Tune 当 Melodyne 用——复音/透明逐音修必须 Melodyne，实时表演必须 Auto-Tune（T02-S014）。
- ❌ 不要堆插件代替"录好源头"——先 mic 选型/摆位/房间，而非 fix-it-in-the-mix。

## 工作流 / Pipeline

> 直接消化 Track 03 的 7 个 workflow。Pipeline：arrangement → tracking → editing → mixing → mastering → delivery + AI/stem 横切。一致性 sanity-check：7/7 workflow 入门 SOP 与资深路径分开、每个含 skip+optimize+add 三类资深差异（远超"≥2 类"门槛）、近期变化字段全填 ✅。每个 workflow 标 last_updated 锚点。资深差异统一用 **跳过 / 优化 / 额外** 三类标注。

### 4.1 Arrangement / Production（把 idea 做到编曲定型）

- **入门 SOP**：①定结构+参考曲（选商业 reference 定 BPM/调/段落，serve-the-song 先想"这歌要什么"）②铺核心四大件（鼓+bass+和声+主旋律建 groove）③编曲填充与动态（能量起伏）④scratch 人声确认留了 negative space ⑤定稿清理（删不服务歌的元素）。每步跳过都有具体后果（漫无目的/满而空/30 秒走神/人声无处安放/带冗余进 tracking）。
- **资深路径**：**跳过**"把每个段落填满"（Rubin/Finneas 式减法，主动删轨；过度编曲是最常见病）；**优化**用自建 session 模板 + 个人音色库/Splice 收藏秒起骨架（trap 用 sample-driven 起点）；**额外**在编曲阶段就为混音预留空间（频段分工，别让 5 件乐器挤 200-500Hz；动态留白）。
- **近期变化**：行业稳态，仅渐进（Splice/AI 生成 loop 降低从零编曲门槛，核心流程 5-10 年稳定）。last_updated: 2026-06-18（Decay risk: low）。evidence: [T03-S009, T03-S010, T03-S001]

### 4.2 Tracking / Programming（录音 + 编程把表演固化成轨）

- **入门 SOP**：①设 gain staging（峰值留 -6~-12 dBFS 余量）+ 监听 ②话筒选型+摆位（U47/SM7B/SM57，先调房间/源头）③录多 take + 录 DI（吉他/贝斯留 reamp 余地）④编程节拍/合成 ⑤命名/标注/整理。跳过后果：录削波/底噪、相位/音色救不回、没 take 可挑、无节奏骨架、下游效率崩塌。
- **资深路径**：**跳过**过度补录（知道哪条 take 是"the one"，源头对了就停）；**优化**把精力前移到源头（话筒/摆位/演奏者状态/房间，Apollo 挂 UAD 监听听到接近成品）；**额外**录音时就留 headroom + 标注 + 保 DI/reamp（初学者推满图"响"，资深留余量给插件甜区+后级）。
- **近期变化**：行业稳态，仅渐进（32-bit 浮点录音让 gain staging 动机从"防内部削波"偏向"插件甜区+心理稳定"）。last_updated: 2026-06-18（Decay risk: low）。evidence: [T03-S001, T03-S002, T03-S010]

### 4.3 Editing（comping / quantize / pitch-correct / time-align）

- **入门 SOP**：①comping（take folder 拼最佳片段）②time-align 对齐（双轨/和声/多话筒鼓相位）③quantize（对齐网格但按需留 swing）④pitch-correct（Auto-Tune 实时音染 / Melodyne 离线透明复音）⑤清理+crossfade+去呼吸噪声。跳过后果：用带瑕疵 take、双轨发虚、节奏飘、走音留成品、爆音杂音。
- **资深路径**：**跳过**"把一切修到完美网格/完美音高"（over-quantize 杀 groove、over-tune 去情绪，刻意留 pocket）；**优化**take folder/rating+快捷键+批处理极快 comp、Melodyne 一次拉一群音；**额外**先听整体 feel 再决定修多少（服务歌的情绪而非指标完美）。
- **近期变化**：渐进无颠覆（AI 自动选 take/Melodyne 复音识别更准；AI stem 分离让"对已混音素材做编辑"成为可能，见 4.7）。last_updated: 2026-06-18（Decay risk: low-medium）。evidence: [T03-S002, T03-S010, T03-S016]

### 4.4 Mixing（gain→balance→EQ→comp→space→automation→bus/mix-bus）

- **入门 SOP**：①gain staging+路由+静态平衡（先让歌在没处理下基本成立，Senior：balance 第一性）②reference A/B（响度匹配后）③EQ 减法优先（清 mud/harsh 再小幅加色）④compression+并行压缩（加厚不压瞬态）⑤space（reverb/delay 建深度）⑥automation（副歌推人声、桥段开混响）⑦bus/mix-bus glue（SSL 总线轻压抱团）+ print。跳过任一都有具体后果（越修越乱/被房间骗/频段打架/动态失控/混音平贴脸/能量丢失/各唱各的）。
- **资深路径**：**跳过**"从空白 session 起步 + 到处加处理"（套自建模板：Scheps 3-bus / CLA 印好的链；默认不动、每个处理可解释）；**优化**reference A/B 贯穿全程 + 结构化总线哲学（3-bus / Brauer multing 把压缩决策模板化，少试错）；**额外**做 mono 兼容 + 多系统/小音箱 + 安静音量复核（初学者只在一对监听大声听）。
- **近期变化**：2024-2025 起由 AI mix assistant + Atmos 推动**混音起点**改变（初学者从 AI 起点接管而非从零；部分项目要求同时交立体声+Atmos）——资深人仍把 AI 当起点、reference A/B+用耳校。last_updated: 2026-06-18（Decay risk: medium）。evidence: [T03-S001, T03-S007, T03-S011]

### 4.5 Mastering（loudness / EQ / limiting / dither / sequencing）

- **入门 SOP**：①校准监听+reference 母带 A/B（响度匹配后）②全局温和 EQ+动态微调（问题尽量回混音解决）③限制器命中平台 target（≈ -14 LUFS streaming，官方/业内共识 T03-S005，到 target 即止不过压）④true-peak ≤ -1 dBTP 余量 ⑤dither（降位深时一次）⑥排序+曲间间隔+整张连续聆听。跳过后果：别处听垮/音色不统一/被平台拉低或牺牲动态/转码削波/量化失真/专辑听感断裂。
- **资深路径**：**跳过**"为更响而过压"（normalization 时代过压只赔动态，主动留动态；也跳过"母带补混音的锅"宁可退回）；**优化**校准监听+整张专辑统一一条母带链、用 reference 母带快速定 target；**额外**管 true-peak 余量 + 一次 dither + 多平台 normalization 预演（"-14 是上限触发点不是硬目标"，官方/业内共识 T03-S006）。
- **近期变化**：streaming LUFS normalization 制度化（已稳定）重写母带目标假设：从"推到 0 dB 最响"转为"留 dBTP 余量、按平台 target 定响度、优先保动态"；BS.1770-5 与 EBU R128 v5.0 均 2023-11 更新；AI 母带（Ozone Assistant/LANDR）自动化"母带起点"，专业发行仍人工主导。last_updated: 2026-06-18（Decay risk: medium）。evidence: [T03-S006, T03-S005, T03-S004]

### 4.6 Delivery（DDP / streaming spec / stems / Atmos）

- **入门 SOP**：①确认目标渠道与规格（采样率/位深/响度/peak/文件类型）②导出主交付文件+响度/peak 合规复检 ③嵌入元数据（ISRC 每条录音唯一码）④按渠道生成专属包（压盘→DDP image+checksum+subcode；客户/sync→stems 分组子混；streaming→平台规格 WAV）⑤交付前最终校验（checksum/试听/命名版本）。跳过后果：格式被打回/转码削波/版税断链/压盘厂无法接收/交错版本返工。
- **资深路径**：**跳过**"母带阶段才想交付规格"（母带前就锁 dBTP 余量与 target，不先做满再削）；**优化**批量/脚本化元数据 + 模板化交付包（一套流程产出 DDP+streaming WAV+stems+备份）；**额外**做多格式/多平台+沉浸式+冗余校验（备好立体声+Atmos ADM BWF，永远 bounce 一条立体声 master 随 ADM BWF 一起交兜底，binaural metadata 设 Near/Mid/Far）。
- **近期变化**：2024-2025 起 Dolby Atmos/沉浸式普及多出**第二条交付链路**（ADM BWF，48kHz/24-bit，分发到 Apple Spatial Audio/Amazon/TIDAL），立体声仍是基线。last_updated: 2026-06-18（Decay risk: medium）。evidence: [T03-S014, T03-S018, T03-S019]

### 4.7 AI / stem-driven 重混 + 起点生成（cross-cutting，近 12 月真正新 workflow）

- **入门 SOP**：①选引擎拆 stem（Logic 本地 / RipX / Moises / RX Rebalance）②伪影 QC（单听每个 stem 找水声/相位/残留，决定能不能用）③（可选）AI mix/master assistant 起点 ④人工接管精修（按 4.4/4.5 流程 reference A/B+用耳改）⑤null/A-B 验证（响度匹配，确认真更好而非"只是不一样"）。跳过后果：引入伪影/伪影进下游被放大/把 AI 输出当成品（最常见错误）/被"变化=改进"骗。
- **资深路径**：**跳过**"盲信 AI 输出"（一律当起点不当终点，有原始多轨时根本跳过 stem 分离）；**优化**按素材选引擎+优先本地（Logic 本地几十秒、隐私好；不只用一个引擎，知道哪种素材哪个引擎伪影最少）；**额外**做伪影 QC + null/A-B + 不让分离 stem 进母带链（用 null test 戳穿"AI 一定更好"的营销）。
- **近期变化**：这个 workflow 本身就是近 12 个月的变化（Logic 11 本地 Stem Splitter、RipX、Moises/Lalal/AudioShake 云引擎质量跃升、AI mix assistant 与 AI 母带成熟）——直接改写"无多轨也能重混/采样" + "混音/母带起点 AI 化"。last_updated: 2026-06-18（Decay risk: high，本 skill 衰减最快处，建议每 3 个月跑一次 update）。evidence: [T03-S016, T03-S020, T03-S017]

## 表达 DNA

> 模拟"这一行的资深人聚一起讨论时的 register"——多人融合，不模仿单个 figure。这一行的 register 跨流派会分裂（见 §7），voice 也应体现：emotion-first 派（Rubin/Mixerman/Scheps）讲"感觉/服务歌/减法"，measurement 派（Worrall/Katz）讲"null test/测出来/dBTP"。

### 5.1 高频用语 / 黑话（听感隐喻为主，印证"用耳不用眼"）

- **听感隐喻黑话**：mud（200-500Hz 浑浊）/ harsh（2-5kHz 刺耳）/ boxy（300-600Hz）/ glue（胶水抱团）/ the pocket（卡在 groove 甜点）/ punch / air / presence / pumping / vibe / mojo。全是听感词，不是参数词。
- **流程/技术词**：gain staging / headroom / serve the song / reference（A/B）/ subtractive（减法）/ 并行压缩（NY compression）/ stems / print/bounce / recall / comping / quantize / humanize。
- **缩写**：LUFS / dBTP（true peak）/ ITB-OTB / DI / M/S / DAW / DDP / ISRC / LRA。

### 5.2 严肃 register（长访谈/conference talk 口气，口语化）

- 资深人在长 podcast/masterclass 里口语化、务实、反 PPT 话术——不说"赋能/闭环/生态"。
- 减法/克制是高地：用"less but knowing why""有时最好的处理是什么都不做""serve the song"作判断锚。
- measurement 派则冷峻精确：用"跑 null test""测出来是什么""dither 只在最后降位深加一次"。

### 5.3 内 vs 外沟通差异

- 同行间：用 mud/glue/pocket/mojo 等隐喻 + 黑话直接判断，默认对方懂频段映射。
- 对非从业者（艺人/客户）：展开成"让歌更清晰/更有情绪""帮你把脑子里那首歌变成现实"，避免术语轰炸。

### 5.4 对话样本库 (industry voice 实战语料) · voice_confidence: medium

> 来源是 Track 01 figures 的 voice_samples（多为可靠转述，少量原话；4 位顶级 figure 本人少发声，故标 medium——见 §8）。每段带 source_id 供回溯。

#### 5.4.1 客户版（面客解释 / 教育）

- 「我把自己当作 reducer 而非 producer——用尽可能少的信息把那个点传达出来。你来找我不是要我加东西，是要我帮你剥到真东西。」(source: T01-S021, 转述自 60 Minutes 2023, 客户场景: 制作方向解释)
- 「我的工作没变——尽可能有创意，帮艺人把脑子里那首歌变成现实。卧室那种紧、亲密、安静的声音，我喜欢它对人声的作用。」(source: T01-S010 / T01-S024, 转述, 客户场景: 录音/制作期望管理)
- 「release-quality 的混音靠的是技术和训练过的耳朵，不是一张昂贵器材清单——先学会该听什么，房间和器材远在其后。给你免费的专业多轨，回家练就是了。」(source: T01-S053, 转述自 RSR455 长访谈, 客户场景: 入门/小工作室期望管理与教育)

#### 5.4.2 同业版（私下 / 内训 / 同业访谈）

- 「不是要做更多，而是要做更少——但你得知道你为什么这么做。我的模板起手没有任何 active 处理，任何会染色的东西都是关着的。」(source: T01-S001 / T01-S035, 转述 Tape Op #133)
- 「我真正的 secret 是听得非常小声——低音量监听才是赢家。开局先把所有推子推上去找到歌在哪，做鼓时先把压缩全摘掉，从无压缩开始平衡。」(source: T01-S037, 转述, 同业技术访谈)
- 「我把混音分送进几个子立体声总线，每个挂不同压缩器——B 给节奏组、C 给中心、A 给高频、D 当额外。multing 是我混 Aretha 时一加贝斯人声就消失逼出来的。」(source: T01-S041, 转述, 同业手法)

#### 5.4.3 监管 / 专业版（公开场合谈标准 / 母带纪律）

- 「响度战争结束了——一旦 normalization 成默认，唯一能更响的办法就是听众自己把音量拧大。用你的耳朵，别盯着一个 LUFS 数字。」(source: T01-S046 / SOS, 转述, 母带标准)
- 「跑 null test——如果残差只是 dither 噪声，那就没有魔法，只有营销。如果它对你来说好听，没人在乎它测出来像不像模拟。」(source: T01-S056, 转述, 反玄学技术立场)

#### 5.4.4 反例版（资深人绝不会这样说 / 被错位包装的厂商话术）

- 「这台插件有真正的模拟 warmth 和 mojo，硬件级染色，必然让你的混音更专业。」(source: T06 厂商话术拒绝, why 反例: 资深人会要求 null test 盲测，warmth/mojo 常是谐波染色/安慰剂，不是"必然提升")
- 「用我们的一键 AI 母带，上传就得到专业母带成品，越响越有冲击力，直接发行。」(source: T06 厂商话术拒绝, why 反例: AI 母带是起点不是终点、需 A/B 原始；"越响越好"是已被 normalization 证伪的响度战争遗毒)

## 质量基准 + 反模式

### 6.1 什么算"好"（可验证基准）

1. **mono 兼容 + 跨系统 translation**：折成 mono 不塌、过手机/车载/耳机/大监听都成立、降到对话音量平衡仍对——而不是只在一对监听大声听好（T03-S001 / T01-S037）。
2. **响度匹配后对 reference 不输**：和同 genre 商业 reference 做 gain-match A/B，音色平衡/清晰度/冲击力站得住，而非靠"更响"显得好（T04-S001）。
3. **频段各就各位、减法清晰**：核心元素各占不同频域、无 mud/harsh/boxy 堆积，清晰度靠减法而非靠推音量（T04-S001 / T02-S001）。
4. **母带响度/交付合规且保动态**：命中平台 target（≈ -14 LUFS）即止、true-peak ≤ -1 dBTP、dither 仅最后降位深一次、Atmos 附立体声兜底（T03-S004 / T03-S018）。
5. **每个处理可解释**：能讲出"这条 EQ/这台压缩为什么在这"，而不是"模板里就有/听起来 pro"——less but knowing why（T01-S001 / T04-S022）。

### 6.2 反模式（外行/入门常犯）

- **comping 当成压缩**（comping=拼 take，compression 才是压缩）。
- **把母带做到 0 dB / 越响越好**（留 true-peak ≤ -1 dBTP，响度战争已被 normalization 终结）。
- **stems 说成"每一轨"**（stems=分组子混，逐轨是 multitracks）。
- **true peak 设到 0 就安全**（inter-sample peak 会越过 0，需 dBTP 余量）。
- **没响度匹配就 A/B reference**（更响的总显得更好，导致误判）。
- **跳过静态平衡直接堆插件**（连基本响度关系都不对，越修越乱）。
- **到处加 EQ 不做减法**（哪里不够就 boost，越加越糊/刺耳）。
- **过度修音出塑料感 / 过度量化丢 pocket**（修没了情绪/groove，in-the-pocket ≠ on-the-grid）。
- **dither 多次 / 浮点链乱加 / 降位深漏加**（只在最后降位深用一次）。
- **拿 AI mix/母带输出直接当成品发**（不 A/B、不用耳朵接管，信营销）。
- **有原始多轨却用 stem 分离**（分离永远有伪影，原始多轨永远更干净）。
- **we'll-fix-it-in-the-mix**（源头随便录指望后期补救）。

## 智识谱系

> 保留分歧而非抹平。本行有四条活跃的流派轴线，各有奠基、代表作与当前代表。

### 7.1 ITB 全数字 ⇄ 模拟硬件 "mojo"

- **ITB 派**：全数字、召回完美、可重复。奠基/当前代表 → **Andrew Scheps**（从顶级模拟棚出身却公开转向全 ITB，3-bus/Rear Bus，T01-S001）；理论锋刃 → **Dan Worrall**（null test 论证大量"模拟魔法"不可闻/是安慰剂，T01-S056）。
- **模拟 outboard 派**：硬件谐波染色"character"不可替代。当前代表 → **Chris Lord-Alge**（"插件给不了硬件那种 character"，坚守 SSL desk + Blue Stripe 1176，T01-S037）、**Jens Bogren** + UA 硬件传统。
- **核心分歧**：mojo/warmth 是真实可闻的谐波染色，还是 level-match 后会抵消的安慰剂。主战场是论坛 null-test thread。这是本行最活的争论，且 Scheps vs CLA 是两位顶级 figure 正面对立的绝佳教学锚点（T02-S020）。

### 7.2 invisible 混音 ⇄ character 混音

- **invisible 派**：不强加可辨识签名，让歌自己说话。代表 → **Serban Ghenea**（"no bag of tricks"，垄断顶级流行混音署名，T01-S007）。
- **character 派**：给歌注入有色彩的个人 sound。代表 → **Tom Elmhirst**（把 vibe "按摩"成 radio-ready，Adele/Amy Winehouse，T01-S044）、**Manny Marroquin**（签名 reverb 用在 80-90% 混音上，T01-S039）。
- **核心分歧**：混音师该不该有可辨识签名（invisible="没风格=不够 artistic" vs character="签名=某种风险/抹平艺人个性"）。

### 7.3 mix by ears / 直觉 ⇄ mix by metrics / 原理（第一性轴 B）

- **emotion-first / 直觉派**：技术服从情绪，用耳不用眼。奠基 → Rubin 减法哲学 + Mixerman《Zen and the Art of Mixing》（"混音首先是决策与意图"）；当前代表 → **Rick Rubin**、**Mixerman (Eric Sarafin)**（T01-S016, T01-S021）。
- **technical-correctness / measurement 派**：校准、测量、第一性原理。奠基 → Katz K-System（JAES 2000）+ AES《Modern Recording Techniques》工程教科书传统；当前代表 → **Bob Katz**（校准监听/K-System，T01-S045）、**Dan Worrall**（反玄学信号处理深挖，T01-S056）。
- **核心分歧**：判断"好"的最终依据是情绪/听感，还是可测量/可复现的指标。两派并非反对方——measurement 派也"use your ears"，emotion-first 派也要懂技术，但优先级不同。

### 7.4 响度战争(峰值最大化) ⇄ dynamics / normalization；及 bedroom 民主化 ⇄ 高端棚

- **响度/动态轴**：历史上的"越响越好"（loud 摇滚混音传统）⇄ 反响度战争（动态优于响度）。当前代表 → **Bob Katz / Bob Ludwig（已退休 2023）/ Greg Calbi**（T01-S046, T01-S047）。开放争议：响度战争是否真结束（Katz 宣称结束 ⇄ Spotify -14 下仍有"迷你响度战"，T04-S008/T04-S021）——保留为未解争议，不假装有标准答案。
- **民主化轴**：bedroom→pro 民主化（免费多轨 + 技术/耳训 > 器材）⇄ 高端棚/付费墙现实。代表 → **Mike Senior**（免费多轨）/ **Finneas**（卧室）/ **Warren Huart**（PLAP）**vs** Larrabee/Electric Lady 大棚现实（Marroquin SSL XL 9000、Elmhirst）（T01-S052, T01-S039）。
- **元争论**：AI 混音/母带是否毁 craft——**Owsinski**（AI 音乐=slop 但 AI 工具可用）/ **Worrall**（测量看待）/ **Huart**（拥抱 AI 工具）（T01-S060）。
