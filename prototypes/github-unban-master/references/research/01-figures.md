- **核心思想关键词**: developer freedom, sanctions compliance, OFAC licensing, platform accessibility, open source universality

## Phase 2 提炼提示

### 反复出现的关键词 (≥ 3 figures 提及)

1. **platform governance / 平台治理** — Lessig, Klonick, Keller, Gillespie, Balkin, Doctorow 均从不同角度论述平台的治理权力. 这是贯穿所有 figure 的核心概念
2. **developer freedom / 开发者自由** — Friedman, Dohmke, Doctorow, Kuhn 均使用该概念, 但含义分化: Friedman/Dohmke 指"让全球开发者使用 GitHub 的权利", Doctorow 指"从平台锁定中退出的自由", Kuhn 指"软件自由 (copyleft 意义)"
3. **sanctions compliance / 制裁合规** — Friedman, Dohmke, Ferrari 直接处理; Keller, Klonick 从法律框架角度间接涉及. 这是该领域最核心的操作场景
4. **over-removal / 过度删除** — Keller, Klonick, Balkin 均讨论平台因法律责任激励而过度删除内容/封禁账号的问题. 直接适用于 GitHub 的"误封"场景
5. **intermediary liability / 中介责任** — Keller, Klonick, Balkin, Lessig 从不同法律传统 (美国 Section 230, EU DSA, 中国网安法) 讨论平台的法律责任边界
6. **code is law / 代码即法律** — Lessig 提出, Doctorow 从 enshittification 角度扩展, Gillespie 从审核实践角度验证. 该概念在 GitHub 场景中高度适用: GitHub 的 IP 检测代码/2FA 策略/制裁筛查算法本身就是"法律"

### 显著分歧 / 流派分裂

1. **合规派 vs 对抗派**: Friedman/Dohmke 代表"在法律框架内争取最大自由"的合规路线; Doctorow 代表"法律本身有问题, 需要通过互操作性和反垄断打破平台权力"的对抗路线; Kuhn 代表"用 copyleft 法律工具直接执法"的第三条路线
2. **平台善治 vs 平台必恶**: Klonick/Gillespie 将平台定位为"新总督/互联网守护者" (带有可改良的暗示); Doctorow 将平台定位为"注定 enshittify 的利润最大化实体" (结构性批判); Keller 介于两者之间 (关注制度设计)
3. **许可实用主义 vs 自由原则主义**: Meeker 代表商业友好的实用主义 OSS 合规; Kuhn 代表 copyleft 强执行的自由原则主义. 两者在 GitHub DMCA/takedown 场景中代表不同的建议路径
4. **全球一体化 vs 区域分裂**: 庄表伟/开源社视角强调中国开源社区融入全球生态; 但 2025 年 GitHub 中国 IP 事件和 Linux kernel 移除俄罗斯维护者事件凸显出"开源无国界"理想与地缘政治现实的裂痕

### 冷僻信号 / 值得关注的边缘观察

1. **Mikeal Rogers 的遗产** — Rogers 2025 年 6 月去世, 但他建立的开源治理模型 (Node.js Foundation → OpenJS Foundation) 仍是理解"平台中立的社区治理"的关键参照. 他的治理模型与 GitHub 的单一平台模式形成对比
2. **1995parham github-do-not-ban-us** — 2019 年伊朗开发者自发组织的抗议 repo (12K+ stars), 是"被封禁开发者集体行动"的范例, 且直接推动了 GitHub 寻求 OFAC license 的进程
3. **Mike Hanley 的角色切换** — GitHub 首任 CSO (2021) → CTO → 2024 年底离职去 GM. 他在 GitHub 的安全策略 (强制 2FA) 直接影响了部分用户的账号可用性 (无法收到短信验证的地区). 其离职后 GitHub 安全策略由谁主导值得追踪
4. **Thomas Dohmke 辞任后的权力真空** — Microsoft 未直接替换 GitHub CEO, 而是将 GitHub 拆分到 Microsoft CoreAI 部门. 这意味着 GitHub 的政策决策从"独立 CEO"模式变为"Microsoft 企业官僚"模式, 对申诉路径可能产生结构性影响
5. **Tarleton Gillespie 的利益冲突** — 平台治理最知名的学者之一在微软 (GitHub 母公司) 工作, 这一身份限制了他对 GitHub 具体治理问题的批评能力
6. **中国 figure 的来源困境** — 庄表伟/适兕/卫剑钒等中国开源社区关键人物的主要输出平台 (微信公众号/知乎/CSDN) 全部在调研黑名单中, 导致可引用的一手材料严重不足. 这不是信号缺失, 而是来源渠道结构性受限
