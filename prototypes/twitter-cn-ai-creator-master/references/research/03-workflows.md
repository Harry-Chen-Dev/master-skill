## Workflows

### WF1. 端到端内容生产 SOP（主干 — 编号步骤 + 易崩点 + 修法）

- **One-liner**: 从「我想发点 AI 内容」到「发布并维护好一条高信噪比推/thread」的完整闭环。所有其他 workflow 都是这条主干的特化。
- **Trigger**: 想发一条值得发的 AI 内容（不是为发而发）。
- **Output**: 一条带来源/带洞察/排版干净/外链不踩坑的推或 thread + 发布后 1-2h 的回复维护。
- **入门 SOP（minimum viable steps）**:
  1. **找信息源**：盯英文一手——arXiv (cs.AI/cs.CL)、AI lab 官博、Hacker News（社区投票=真热度）、Reddit；用 Folo/Feedly 把源收口到一条 AI 摘要时间线 (evidence: [T02-S015, T02-S017, T02-S018])。
     - 跳过后果：只能搬别人的中文二手稿 → 无信息差、踩平台搬运红线、2026 被判 aggregator 削分成 (evidence: [T03-S003])。
  2. **选题（信息差/热点/价值判断）**：选「英文已热、中文还没人讲清」的；判断标准 = 信息差 ×（受众价值 + 时效）。HN 首页/arXiv 热度是热度信号 (evidence: [T02-S017])。
     - 跳过后果：选题撞车或自嗨，无人转无人回。
  3. **编译/总结/加洞察**：让 AI **理解后用中文重写（不是逐句翻译）**，再人工加本土化观点。这是质量分水岭（详见 WF2）(evidence: [T02-S011, T01-S001])。
     - 跳过后果：出「机翻味」，读者一眼识破搬运。
  4. **写推/thread**：首条 = hook（回答「我为什么不划走」）；信息密度（每条要么教点东西要么推进）；短句 + 断行排版 + 留白 (evidence: [T04-S011, T03-S010])。
     - 跳过后果：第一屏没抓住=后面再好也没人读（hook 决定 80-90% 成败，evidence: [T03-S002]）。
  5. **配图**：ray.so/carbon 做代码/金句卡；CleanShot/Xnapper 截 AI 产品 UI/对比图/数据图（截图先脱敏）(evidence: [T02-S006, T02-S009])。
     - 跳过后果：纯文字推触达低；算法对多媒体内容加权 (evidence: [T01-S008])。
  6. **发布（时机 + 外链位置）**：用 Typefully 排到高峰时段（工作日 8-10AM / 12-1PM 受众时区，周三 9AM 最强）；**外链放首推的自回复第二条，正文自足** (evidence: [T03-S001, T02-S023])。
     - 跳过后果：非 Premium 账号外链推近乎 0 曝光（2026-03）(evidence: [T03-S001])；时效衰减 ~6h 减半、24h 后基本无推送 (evidence: [T03-S001])。
  7. **互动维护**：发后 1-2h 蹲守，**回复每一条评论**（回复=27× 赞，作者回复对话=150× 赞，是算法最高杠杆）(evidence: [T03-S001, T04-S003])。
     - 跳过后果：放弃了最高权重的正信号，错过把一条推推爆的窗口。
- **资深路径**：
  - **optimize**（step 3+4）：资深人把「重写 + hook」当核心，一半时间花在前 280 字；用 Tweet Hunter 找 hook 结构（参考不照搬）(evidence: [T02-S019, T03-S002])。
  - **add**（step 6+7）：资深人**默认外链放自回复**（条件反射，不再纠结政策反复）；发后主动 reply 引导对话（埋问题 / 求反例），刻意制造「作者回复」高权重信号 (evidence: [T03-S001])。
  - **add**（step 1）：资深人有自建一手源管线（RSS + 关注英文区关键人物），不靠中文聚合号二手。
- **近期变化**：
  - **2026-01 起**：X 用 **Grok 驱动的 transformer 排序模型**取代旧推荐系统，会「读每条推、看每个视频」做匹配——内容语义质量权重上升，标题党/薄搬运更易被压 (evidence: [T03-S001, T03-S011, T03-S012])。
  - **2026-03 起**：非 Premium 外链推近乎 0 中位互动 (evidence: [T03-S001])。
  - 触发类型：新模型(Grok 排序) + 平台政策。
- **典型耗时**：入门一条 thread 2-4h；资深 0.5-1.5h（管线熟 + AI 辅助）。
- **关键工具**：Folo/arXiv/HN（选题，必备）；Claude/Kimi/DeepSeek（重写，必备）；ray.so/截图（配图，必备）；Typefully（写+排，必备）；X Premium（长推/编辑，必备）。
- **关键人物**：宝玉 @dotey（编译三阶段，T01-S001/S003）；歸藏 @op7418（算法顺应+多回复，T01-S008）。
- **常见失败模式**：
  - 让 AI 直接「翻译」→ 机翻味被识破。修：理解后中文重写 + 人工加洞察。
  - 外链放首推 → Non-Premium 近乎 0 曝光。修：外链永远放自回复第二条。
  - 发完即走不回复 → 丢掉 27×/150× 高权重信号。修：发后蹲守 1-2h 回复。
- **Last_updated**: 2026-06-06
- **Decay risk**: high（Grok 排序 + 外链/变现政策都在 12 月级变动）

### WF2. 编译一手 → 中文推（招牌打法：翻译即重写 + voice 注入）

- **One-liner**: 中文 AI 圈最高杠杆打法——把英文一手做成「带个人 voice 的深度搬运」，跨越「机翻搬运」和「原创洞察」之间。
- **Trigger**: 抓到一篇英文一手（论文/高管访谈/官博/视频字幕）值得搬给中文受众。
- **Output**: 一条/一串读起来像「中文原生写的、还带作者观点」的编译推，非逐句译。
- **入门 SOP**:
  1. **拿一手全文**（不是别人的中文摘要）(evidence: [T02-S015])。
  2. **AI 理解 + 中文重写**：宝玉式三阶段——① 多角色 prompt（一个英文老师直译 → 两个中文老师按不同风格意译 → 校长融合）；② Agent 阶段先分析术语/文化隐喻/背景知识；③ 人工在关键节点确认 (evidence: [T01-S003, T01-S004])。
  3. **过一道「写作风格 Skill」注入个人 voice**：让产出带上你的表达习惯、注释方式、读者意识——这是区别于纯机翻搬运的关键 (evidence: [T01-S001])。
  4. **人工加一层洞察/观点**：编译之上加「这对中文读者意味着什么 / 我的判断」，否则就是高级搬运 (evidence: [T02-S259, T01-S001])。
  5. **标来源**：注明英文原文出处（信任 + 避 2026 aggregator 削分成）(evidence: [T03-S003])。
- **资深路径**：
  - **add**（step 3+4）：资深人有固化的 voice skill / prompt 库（如歸藏/宝玉公开的 GitHub skill 仓库），且**每篇都加原创判断**，不只搬 (evidence: [T01-S006, T04-S013])。
  - **optimize**（step 2）：长文用 Kimi（200 万字上下文）一次喂全文做总结，快讯用 DeepSeek（中英对译强、性价比高）(evidence: [T02-S011])。
- **近期变化**：
  - **2026 起** X 测试「识别原创作者」工具，把分成更多导向原创、削减 aggregator（纯搬运/薄改写）账号——本周期 aggregator payout 降到 60%（-40%），下周期再 -20% (evidence: [T03-S003])。**含义**：纯翻译搬运的变现空间被挤压，「重写 + 加洞察 + 标来源」从「质量选择」变成「保命动作」。
  - 触发类型：平台政策（反 clickbait/搬运）。
- **典型耗时**：入门 1-2h/篇；资深 30-60min（prompt 链熟）。
- **关键工具**：Claude/GPT（语感）、Kimi（长文本）、DeepSeek（快译）；宝玉/歸藏公开 skill 仓库。
- **关键人物**：宝玉 @dotey（招牌，T01-S001/S003/S004）；向阳乔木 @vista8（半自动改写流水线 QMReader，T01-S010）。
- **常见失败模式**：
  - 让 AI「翻译」而非「重写」→ 机翻味。修：多角色重写 prompt。
  - 只搬不加洞察 → 2026 被判 aggregator 削分成 + 无差异化。修：每篇必加一层个人判断。
  - 不标来源 → 信任崩 + 踩搬运红线。修：固定来源注明模板。
- **Last_updated**: 2026-06-06
- **Decay risk**: high（变现政策 + 模型能力双变量）

### WF3. thread 写作流（首条钩子 → 编号逻辑链 → 结尾 CTA）

- **One-liner**: 把一个论点拆成多条相连推，首条 hook 决定生死、中间一推一念、末条引导关注。
- **Trigger**: 一个想法值得展开成 3+ 条推（atomic essay 装不下）。
- **Output**: hook→逻辑链→CTA 的完整 thread。
- **入门 SOP**:
  1. **写首条 hook**：回答「我为何不划走」。用 hook 公式（见下）。花一半写作时间在这一条 (evidence: [T03-S002, T04-S010])。
  2. **拆逻辑链**：每条推一个念头（one idea per tweet），编号串联，前一条留缺口逼读下一条（curiosity gap）(evidence: [T03-S010, T04-S011])。
  3. **排版**：短句 + 断行 + 留白，让 thread 可 skim；适当配图/表 (evidence: [T03-S010])。
  4. **末条 CTA**：引导关注/转发/收藏——必须是价值的自然延伸，不硬塞 (evidence: [T04-S011])。
  5. **外链放末条的自回复**（如有）(evidence: [T02-S023])。
- **单条爆款 hook 公式（一手，6 类）** (evidence: [T03-S002, T04-S011])：
  | 公式 | 模板/例 |
  |------|--------|
  | 内幕清单（Insider listicle） | 「你被瞒着的 X 件事…」+ 列表 |
  | 故事框架（Story） | 给一个时间点 + 结局，留「怎么做到的」悬念 |
  | 点名受众 + CTA（Specificity） | 「如果你是 20 多岁做 AI 的人，这条为你写」 |
  | 权威 + 大数字（Authority+Numbers） | 「我读了 200 篇论文，提炼成 5 个框架」 |
  | 专家综合（Synthesis） | 「研究 X 多年，distill 成 Y 个框架」 |
  | 反直觉观点 + 痛点/解法（Contrarian） | 「大多数 AI 涨粉建议是错的，真正有用的是…」 |
  - 补充开场技巧：强陈述句 / 发人深省的问句 / 脆弱自白 / 怪异独特洞察 / 具体数字+时间框架（「9 周减 23 磅」胜「快速减肥」）(evidence: [T04-S011, T03-S002])。
- **资深路径**：
  - **optimize**（step 1）：资深人写 5-10 个 hook 变体再选；用 Tweet Hunter/爆款库找结构（参考不照搬）(evidence: [T02-S019])。
  - **add**（step 2）：资深人确保「一推一念」+ 每条能独立被引用/截图。
- **近期变化**：2026-01 Grok 排序读全文 → 逻辑连贯、信息密实的 thread 比堆 hook 的水 thread 更吃香 (evidence: [T03-S001])。触发：新模型。
- **典型耗时**：入门 1.5-3h；资深 45-90min。
- **关键工具**：Typefully（分条编辑器）、Tweet Hunter（hook 灵感）。
- **关键人物**：Dickie Bush/Nicolas Cole（Ship30 hook，T03-S002）；王川 @Svwang1（深度长 thread 参照，T01-S020）。
- **常见失败模式**：
  - hook 平庸（「关于增长的一些思考」）→ 没人点开。修：用具体数字/反直觉公式。
  - 一条塞多个念头 → 读者掉队。修：one idea per tweet。
  - CTA 硬推关注 → 油腻。修：CTA = 价值延伸。
- **Last_updated**: 2026-06-06
- **Decay risk**: medium（hook/结构原理稳定，排序模型影响中等）

### WF4. 不同内容类型工作流（快讯 / 深度长推 / 锐评 quote / build-in-public）

> 四条特化链路，共用 WF1 主干但各有取舍。

#### WF4a. 快讯编译流（抢时效）
- **Trigger**: 新模型/产品刚发布，要第一时间出中文速递。
- **Output**: 「产品名 + 一句定位 + 功能 bullet + 来源链接」清单体快讯 (evidence: [T01-S014])。
- **SOP**: Folo/RSS 抓一手 → DeepSeek/Kimi 快速理解+中文重写 → 截图美化配图 → Typefully 发，外链放自回复 (evidence: [T02-S240])。
- **资深差异**：`skip` 深度展开（速度优先）；`optimize` 推文模板化（小互式功能清单体，便于快速消费）(evidence: [T01-S014])；`add` 仍标来源 + 加一句独有判断（防 2026 aggregator 削分成）(evidence: [T03-S003])。
- **失败模式**：全自动批量发不核验 → 低质同质、被判内容农场/aggregator (evidence: [T01-S021, T03-S003])。修：聚合 agent 只做选题前置，发前人工核验。
- **Decay risk**: high（聚合层正被全自动吞噬 + aggregator 削分成）。

#### WF4b. 深度长推 / 论文速读
- **Trigger**: 一篇论文/长文要讲透。
- **Output**: Premium 长推（≤25000 字）或 Articles，绕开外链降权（内容留站内）(evidence: [T04-S004])。
- **SOP**: arXiv/官博一手 → Kimi 喂全文总结 或 Claude/GPT 起草（语感+洞察）→ Typefully 分条 → Premium 长推/Articles 承载。
- **资深差异**：`optimize` Kimi 长文本一次喂全文；`add` **前 280 字必须能独立做 hook**（超出后折叠在 Show more）(evidence: [T04-S004])。
- **失败模式**：把深度内容塞外链博客 → 外链降权。修：用站内长推/Articles。
- **Decay risk**: medium。

#### WF4c. 锐评 quote / build-in-public 帖
- **Trigger**: 有原创观点，或有产品进度/数据可晒。
- **Output**: 锐评推（quote 转发带评论）/ 复盘帖。
- **SOP**: Claude/GPT 帮打磨语感与论证（**观点必须是你自己的**）→ 发；build-in-public 则晒进度/数据/挣扎 (evidence: [T01-S015, T04-S009])。
- **资深差异**：`add` 数据透明复盘（Orange AI 式「172 万曝光→309 订阅，比例千分之二」公开转化漏斗）(evidence: [T01-S015])；`skip` AI 全代笔（同质无人味）。
- **失败模式**：AI 代笔锐评 → 千篇一律、失去声音。修：AI 只打磨，观点自己出。
- **Decay risk**: medium。

- **Last_updated**: 2026-06-06

### WF5. 发布前质检环（checklist）

- **One-liner**: 发布前最后一道关，挡掉机翻味/无来源/外链踩坑/弱 hook/AIGC 问题。
- **Trigger**: 任何内容发布前。
- **Output**: 过 checklist 的成品。
- **入门 SOP（逐项 yes/no）**:
  1. **机翻味自检**：读一遍，有没有「英语腔」「直译痕迹」？有 → 回 WF2 重写 (evidence: [T02-S011])。
  2. **来源标注**：编译/引用是否注明英文一手出处？(evidence: [T02-S015, T03-S003])。
  3. **外链位置**：外链是否在自回复而非首推？(evidence: [T02-S023, T03-S001])。
  4. **hook 够不够**：首条/前 280 字能否让人不划走？读不下去 → 重写 hook (evidence: [T03-S002])。
  5. **AIGC 标注/核实**：AI 生成的数字/链接是否核对过（AI 会编造）？AIGC 内容是否合规标注？(evidence: [T02-S269])。
  6. **配图不撞脸**：是否全是 carbon/ray.so 默认渐变？混入截图/对比图区隔 (evidence: [T02-S271])。
- **资深路径**：`add` 资深人把这 6 项内化成发布前条件反射；`optimize` 用 Typefully 预览 + 数据回看迭代。
- **近期变化**：2026 Grok 排序 + aggregator 削分成使「来源标注 + 加洞察」从可选变必做 (evidence: [T03-S001, T03-S003])。
- **典型耗时**：2-5min。
- **常见失败模式**：跳过质检直接发 → 机翻味/编造数字/外链 0 曝光。修：固定 checklist。
- **Last_updated**: 2026-06-06
- **Decay risk**: low（质检项稳定，individual 阈值随政策微调）

### WF6. 起号涨粉 0→1 + build-in-public 增长飞轮

- **One-liner**: 从 0 粉到有垂直受众，再用公开构建做增长飞轮。
- **Trigger**: 新号或想加速增长。
- **Output**: 垂直受众 + 自我强化的增长飞轮（内容→受众→信任→变现→更多内容）。
- **入门 SOP（0→1）**:
  1. **垂直定位**：选一个细分（如 AI 编程 / AIGC 视觉 / 论文速读），别什么都发 (evidence: [T03-S008])。
  2. **英文区信息差红利**：编译英文一手做高信噪比内容（中文圈起号最快的红利，详见 WF2）(evidence: [T01-S002, T01-S012])。
  3. **主动互动**：去目标受众/同行推下高质量回复——回复=27× 权重，且 out-of-network 曝光入口 (evidence: [T03-S001, T03-S007])。
  4. **蹭热点**：新模型发布日抢时效快讯（WF4a），借热度被发现 (evidence: [T03-S008])。
  5. **持续日更**：2-5 条/天，至少 1 条维持算法存在感（注意 creator diversity cap：同一 follower 的 For You 里你的推有上限，狂发反而稀释）(evidence: [T03-S001, T03-S011])。
- **build-in-public 飞轮（资深 add）**:
  - 边做产品/项目边公开晒进度/数据/挣扎 → 真实故事比包装营销更吸引同行 → 你的工作自动产出内容（无需想选题）→ 受众积累 → 成首批用户/订阅 → 反哺更多内容。Linear 把 Twitter 当公开 changelog 是经典案例 (evidence: [T03-S009, T03-S007, T04-S009])。
  - 变现飞轮：thread → newsletter/付费周刊 → 社群/课程/产品（歸藏付费周刊、小互社群日报、花生卖课）(evidence: [T01-S009, T01-S014, T01-S017, T04-S008])。
- **资深路径**：
  - **add**：build-in-public 叙事 + 跨平台矩阵分发（X→播客→微博→即刻，歸藏式重复曝光）(evidence: [T01-S009])。
  - **optimize**：用 Hypefury auto-retweet/evergreen 复用爆款 + auto-plug 在推火后自动挂链接（注意别过度自动化油腻）(evidence: [T02-S002, T02-S003])。
  - **争议起号（高风险变体，对照）**：花生式主动用对立话术制造讨论引流——流量快但人设风险高，谨慎 (evidence: [T01-S017])。
- **近期变化**：
  - **2026-01 Grok 排序** + **creator diversity cap**：狂发频率不再线性提升触达，质量 + 对话 > 数量 (evidence: [T03-S001, T03-S011])。
  - **2026 变现门槛**：≥500 verified followers + 近 90 天 ≥500 万 verified（Premium 用户）主页 timeline 曝光 + Premium 订阅 + Stripe；**回复曝光不再计入 payout，仅主页 timeline 曝光计费**（这是对 2024「回复多=赚最多」的反转——回复仍是涨粉/算法最高杠杆，但不再直接变现）(evidence: [T03-S004, T03-S005, T03-S006])。
  - 触发类型：新模型(Grok) + 平台政策(变现规则改制)。
- **典型耗时**：0→1k 粉数月（垂直+日更+互动）；飞轮见效 6-12 月。
- **关键工具**：Typefully（写排）、Hypefury（增长自动化）、Premium（变现门槛）。
- **关键人物**：Orange AI @oran_ge（build-in-public 复盘，T01-S015）；歸藏（矩阵分发+付费周刊，T01-S009）；Pieter Levels（build in public 原典，T04-S009）。
- **常见失败模式**：
  - 什么都发不垂直 → 算法/受众都抓不住定位。修：选一个细分死磕。
  - 狂发刷屏 → diversity cap 稀释 + 油腻。修：质量+对话优先，2-5 条/天。
  - 把变现寄望于回复曝光 → 2026 已不计入 payout。修：变现靠主页 timeline 曝光 + 订阅/社群/课程。
- **Last_updated**: 2026-06-06
- **Decay risk**: high（算法+变现政策 12 月级变动）

## AI 辅助 vs 全自动（核心张力，保留矛盾）

- **AI 当重写助手（人工精选派，主流共识）**：一手 → AI 理解后中文重写 → **人工加洞察** → 精修配图。慢但有信任、差异化、个人 voice；2026 在 aggregator 削分成 + Grok 读全文下**结构性占优** (evidence: [T02-S259, T03-S003, T03-S001])。
- **AI 当代笔机 / 全自动批量（聚合派）**：聚合 agent + AI 批量生成 + 自动排程（justlovemaki 式 Gemini 自动日报）。量大省时，但低质同质、易被判内容农场，**2026 aggregator payout -40%（下周期再 -20%）+ Grok 压薄搬运** → 代价急剧上升 (evidence: [T01-S021, T03-S003, T03-S012])。
- **半自动居中**：向阳乔木 QMReader「AI 翻译+按风格改写+人工审」——可放量但保人工关 (evidence: [T01-S010])。
- **结论**：**「AI 是重写助手不是代笔机」是 2026 的安全线**。全自动只适合纯快讯且必须人工抽检；护城河已从「聚合速度」上移到「观点/实测/人设/voice」(evidence: [T01-S021])。

## Phase 2 提炼提示

**反复出现 ≥3 个 workflow 都包含的步骤（候选 playbook 通则）**:
- **「AI 重写而非翻译 + 人工加洞察」** 出现于 WF1/WF2/WF4a/WF4b → playbook：「遇到英文一手，让 AI 理解后中文重写并注入 voice + 加一层判断，绝不逐句译」。
- **「外链放自回复、正文自足」** 出现于 WF1/WF3/WF4a/WF4b → playbook：「想放链接 → 首推纯文字钩子，链接放第二条回复」。
- **「hook/前 280 字自足、花一半时间在首条」** 出现于 WF1/WF3/WF4b → playbook：「不管多长，第一屏决定生死」。
- **「发后蹲守回复、制造作者回复对话」** 出现于 WF1/WF6 → playbook：「优先制造能引发回复/作者再回复的对话（27×/150×），而非求赞」。
- **「标来源 + 加原创层」** 出现于 WF2/WF4a/WF5 → playbook：「编译必标英文出处 + 加个人判断（2026 aggregator 削分成保命动作）」。

**入门 SOP 与资深路径最大差距（候选心智模型）**:
- 入门与资深步数接近（均 5-7 步）——这一行资深差异**不在少做步，而在每步的质量动作**（重写深度 / hook 打磨 / 主动对话 / 加洞察）。
- 推断心智模型：**「护城河上移」**——聚合/速度被自动化吞噬，资深人把精力压到机器做不了的「观点/实测/voice/对话」上。
- 推断心智模型：**「对话 > 广播」**——reply 27× / 作者回复 150× / 起号靠主动回复，资深人写推先问「这会引发回复吗」(evidence: [T03-S001, T04-S003])。

**近期工作流变化的根本原因（触发事件类型分布）**:
- **新模型驱动**：2026-01 Grok transformer 排序读全文（影响 WF1/WF3/WF6）。
- **平台政策驱动**：外链近 0 曝光(2026-03)、aggregator 削分成、回复曝光不计入 payout、creator diversity cap（影响 WF1/WF2/WF4a/WF6）。
- 主要外部驱动力 = **平台算法/变现政策反复 + LLM 能力升级**双引擎 → 本 skill 工作流节衰减最快，强烈建议每 3 月跑 update。

**保留的矛盾（Phase 1.5 用户裁决 / 诚实边界）**:
- **外链降权**：2023 罚 → 2025 传闻取消(8× lift, T04-S014) → 2026 非 Premium 近乎 0 曝光(T03-S001)。写作姿势取最稳：外链放自回复。
- **回复与变现**：2024「回复多=赚最多（广告在回复里）」(T04-S006) → 2026「回复曝光不再计入 payout，仅主页 timeline 计费」(T03-S004/S005)。判断：回复仍是**涨粉/算法**最高杠杆，但不再是**直接变现**杠杆——两件事分开。

**冷僻 / 信号薄弱**:
- workflow 数 8 ✓（≥4）；资深差异点每个 workflow ≥2 ✓；近期变化字段 100% 填 ✓。
- **一手 source 占比偏低**：新增 12 源中 verified_primary 仅 2（X 官方 help + Ship30），其余 secondary——因这一行 SOP 的**中文一手复盘绝大多数在黑名单（知乎/公众号）**。靠 seed 的 T01/T02/T04 verified（个人站/GitHub/X 官方）+ 多 secondary 交叉 + 保留矛盾补足。**Phase 2.8 诚实边界须注明**：workflow 维度的中文一手厚度低于英文写作 canon，部分 SOP 步骤从可观察内容行为反推（推断）；且 6 个月内大概率因算法/政策变动部分过时。
