## 详细条目

### 📖 1. The Design of Everyday Things (DOET)

- **Author**: Don Norman（认知科学家，「user experience」一词的普及者，前 Apple UX Architect）
- **Year**: 1988 (原名 *The Psychology of Everyday Things*) / Revised & Expanded Edition: 2013
- **Type**: popular science / 根基性著作
- **One-liner**: UX 行业的「圣经」——把认知心理学变成可操作的设计语汇。 (evidence: [T04-S001, T04-S021])
- **核心论点 (3-5)**:
  1. 用户出错多半是**设计的错**，不是用户的错（怪罪用户是设计失败）。
  2. **affordance（可供性）+ signifier（意符）+ mapping + feedback + 约束 constraints + conceptual model** 是好设计的可分析维度。
  3. 「人因 + 情感」：好用之外，*Emotional Design* 延伸出「attractive things work better」（美的东西被感知为更好用）。
  4. 设计应让正确操作显而易见、错误操作难以发生（forcing functions / poka-yoke）。
- **读完得到什么**: 一套**诊断任何界面/物体好坏的通用词汇**，能说清「为什么这个门/这个 App 难用」。
- **难度**: 入门，但概念是全行业根基。
- **Endorsement evidence**:
  - `[type: course_syllabus]` 几乎所有 HCI/UX 大学课程 + NN/g 培训的指定/推荐读物 (evidence: [T04-S021])
  - `[type: figure_long]` Norman 本人持续在 jnd.org / NN/g 阐述书中概念 (evidence: [T04-S001])
  - `[type: conf_citation]` affordance/signifier 是 UX 会议/文章的默认引用词汇 (evidence: [T04-S004])
- **是否被新版 supersede**: 2013 修订版引入 signifier 概念、补 HCD，**应读 2013 版**（1988 老版仍有教学价值但被覆盖）。
- **替代品**: *Emotional Design* (Norman, 2004) 补情感维度；*The Design of Future Things* 补自动化。
- **如果只读 1 章**: 第 1 章 "The Psychopathology of Everyday Things"（affordance/signifier/mapping/feedback 全在此）。
- **可信度**: high。**Decay risk**: low（概念 35 年稳定）。

### 📖 2. Don't Make Me Think (DMMT)

- **Author**: Steve Krug（可用性顾问）
- **Year**: 2000 / Revisited 3rd Edition: 2014（加入移动端）
- **Type**: practitioner handbook
- **One-liner**: 用最少时间把「可用性常识」装进脑子的入门书。 (evidence: [T04-S003])
- **核心论点**:
  1. **第一定律：「Don't make me think」**——界面应自我解释，认知负荷越低越好。
  2. 用户**不读、只扫（scan）**；满意即止（satisficing），不找最优。
  3. **折扣可用性测试（DIY usability testing）**：自己花一上午、几个用户、出声思考，就能发现大问题——不必等大预算正式研究。
  4. 设计无谓争论时，「测试一个早晨胜过专家争一个月」。
- **读完得到什么**: 能立刻上手做轻量可用性测试 + 一套界面清晰度的检查直觉。
- **难度**: 入门（最快回报的一本）。
- **Endorsement evidence**:
  - `[type: course_syllabus]` UX bootcamp / 大学入门课的标配首读 (evidence: [T04-S021])
  - `[type: figure_short]` 行业广泛公认的「第一本该读的 UX 书」 (evidence: [T04-S003])
  - `[type: blog_secondary]` NN/g 可用性测试方法与之高度一致 (evidence: [T04-S005, T04-S021])
- **替代品**: *Rocket Surgery Made Easy*（Krug 自己的测试操作续作）。
- **如果只读 1 章**: 第 5 章「Omit needless words」+ 测试章。
- **可信度**: high。**Decay risk**: low。

### 📖 3. About Face: The Essentials of Interaction Design

- **Author**: Alan Cooper, Robert Reimann, David Cronin, Christopher Noessel（Cooper = 「VB 之父」/ 交互设计先驱）
- **Year**: 1995 / 4th Edition: 2014
- **Type**: textbook
- **One-liner**: 交互设计的系统性教科书，**persona 与 goal-directed design 的方法论根源**。 (evidence: [T04-S006])
- **核心论点**:
  1. **goal-directed design**：从用户**目标**（而非任务/功能）出发设计。
  2. **persona** 作为设计工具：用研究合成的具体虚构用户对齐团队决策。
  3. 交互设计有自己的原则/模式/惯用法体系（不只是视觉）。
  4. 软件应表现得像「有礼貌、体贴的人」。
- **读完得到什么**: 系统的交互设计方法论 + persona/scenario 的正确用法。
- **难度**: 进阶（厚、体系化）。
- **Endorsement evidence**:
  - `[type: course_syllabus]` 交互设计课程的核心教科书 (evidence: [T04-S006])
  - `[type: figure_long]` Cooper 公司方法论直接来自此书 (evidence: [T04-S006])
  - `[type: conf_citation]` persona 概念被全行业引用 (evidence: [T04-S021])
- **替代品 / 配套**: *The Inmates Are Running the Asylum*（同作者，论战式起源，更易读）。
- **如果只读 1 章**: persona / goal-directed 方法章。
- **可信度**: high。**Decay risk**: low-medium（4 版 2014，部分案例略旧）。

### 📖 4. The Inmates Are Running the Asylum

- **Author**: Alan Cooper
- **Year**: 1999 / 2004 重印
- **Type**: argument / popular
- **One-liner**: 「为什么高科技产品把我们逼疯」——persona 与交互设计师角色的论战起源。 (evidence: [T04-S006])
- **核心论点**: 工程师为自己（而非用户）设计 → 产品难用；解药是引入专职交互设计师 + persona 方法。
- **读完得到什么**: 理解「为什么需要 designer」的论证 + persona 思想的来由。
- **难度**: 入门。**Endorsement**: `[type: figure_long]` Cooper 方法论原点 (evidence: [T04-S006]); `[type: blog_secondary]` 被广泛引为 persona 起源 (evidence: [T04-S021])。
- **可信度**: high。**Decay risk**: medium（技术案例旧，思想不旧）。

### 📖 5. Inspired: How to Create Tech Products Customers Love

- **Author**: Marty Cagan（Silicon Valley Product Group; eBay/Netscape 产品高管）
- **Year**: 2008 / 2nd Edition: 2017
- **Type**: practitioner handbook
- **One-liner**: 现代「产品发现 (product discovery)」的圣经——designer 与 PM、eng 的协作三角。 (evidence: [T04-S007])
- **核心论点**:
  1. **discovery vs delivery** 双轨：先验证「该不该做」再做。
  2. **product trio**（PM + designer + tech lead）共同负责发现。
  3. 反对「功能工厂」(feature factory)——按 outcome 而非 output 衡量。
  4. designer 是产品成功的合伙人，不是出图的「服务台」。
- **读完得到什么**: designer 如何嵌入现代产品组织、影响「做什么」而非只「怎么画」。
- **难度**: 进阶。
- **Endorsement evidence**:
  - `[type: figure_long]` Cagan 本人 + SVPG 系列的核心 (evidence: [T04-S007])
  - `[type: course_syllabus]` 产品/UX leadership 必读 (evidence: [T04-S024])
  - `[type: blog_secondary]` 与 Torres 的 continuous discovery 互补引用 (evidence: [T04-S010])
- **替代品 / 续作**: *Empowered* (Cagan, 2020, 谈赋能团队)。
- **如果只读 1 章**: product discovery 章。**可信度**: high。**Decay risk**: low-medium。

### 📖 6. Refactoring UI

- **Author**: Adam Wathan & Steve Schoger（Tailwind CSS 团队）
- **Year**: 2018（稳定，无重大新版）
- **Type**: practitioner handbook / 视觉速成
- **One-liner**: 给「没有视觉背景」的人一套可立即套用的 UI 美化规则。 (evidence: [T04-S012])
- **核心论点**: 间距/层级/对比/颜色/字重的**具体可操作技巧**（如「用字重和颜色替代字号做层级」「先设计困难/极端状态」）。
- **读完得到什么**: 工程师/产品人也能产出像样界面的清单式技巧。
- **难度**: 入门。
- **Endorsement evidence**:
  - `[type: blog_secondary]` 工程/设计社群广泛推荐为「视觉速成最佳」 (evidence: [T04-S012])
  - `[type: figure_short]` design engineer 圈层常推 (evidence: [T04-S024])

### 💡 Core Concepts 详解（按 tier）

> tier-1 = 所有从业者必懂；tier-2 = 资深者熟知。来源字段追溯首次提出者。

#### tier-1（核心 12）

1. **Affordance 可供性** — `tier-1`。物体属性暗示的可能操作。来源：[primary] J.J. Gibson（生态心理学）→ Norman 引入设计 (DOET, 1988)。关联：signifier/constraint。**演化**：Norman 后来澄清设计师真正能控制的是 *signifier*（可供性是关系不是属性）。**常见误用**：把「视觉提示」直接叫 affordance（其实那是 signifier）。 (evidence: [T04-S001, T04-S004])
2. **Signifier 意符** — `tier-1`。明示「在哪/怎么操作」的可感知线索。来源：Norman (DOET 2013 修订版引入)。关联：affordance。**为什么进 canon**：解决「可供性存在但用户看不见」问题。 (evidence: [T04-S001])
3. **Mental model 心智模型** — `tier-1`。用户脑中对系统运作的认知。来源：Norman；Jakob's Law 直接建立其上。**常见误用**：以设计师自己的心智模型代替用户的。 (evidence: [T04-S001, T04-S004])
4. **Usability 可用性** — `tier-1`。Nielsen 5 维：learnability/efficiency/memorability/errors/satisfaction。来源：Nielsen。 (evidence: [T04-S004])
5. **Heuristic evaluation 启发式评估** — `tier-1`。用 10 启发式做专家走查。来源：Nielsen 1990/1994。关联：usability testing（互补，不互替）。 (evidence: [T04-S004])
6. **Persona 用户画像** — `tier-1`。基于研究合成的虚构典型用户。来源：[primary] Cooper (*Inmates*, 1999; *About Face*)。**常见误用**：拍脑袋造无研究支撑的「假 persona」/塞人口统计。 (evidence: [T04-S006])
7. **Information Architecture (IA) 信息架构** — `tier-1`。内容的组织/标签/导航/搜索结构。来源：[primary] Rosenfeld & Morville *Information Architecture* (1998, "Polar Bear Book")。关联：card sorting/tree testing。 (evidence: [T04-S021])
8. **Wireframe / Fidelity 线框/保真度** — `tier-1`。lo-fi→hi-fi 原型精度阶梯。来源：实践共识。**常见误用**：过早做 hi-fi 导致 stakeholder 纠结视觉而非流程。 (evidence: [T04-S003, T04-S021])
9. **Design system 设计系统** — `tier-1`。组件+token+规范+原则的复用体系。来源：Frost (Atomic Design) / Curtis。 (evidence: [T04-S012])
10. **Double Diamond / Design Thinking / Design Sprint（流程三件套）** — `tier-1`。问题空间↔方案空间的发散收敛流程。来源：UK Design Council / IDEO-d.school / Knapp。 (evidence: [T04-S023, T04-S016, T04-S008])
11. **JTBD（Jobs To Be Done）** — `tier-1`。用户「雇用」产品取得的进步。来源：[primary] Christensen (HBR 2016) + [significant follow-up] Ulwick ODI / Moesta 访谈法。 (evidence: [T04-S017])
12. **Accessibility (a11y) / WCAG** — `tier-1`。无障碍设计与 W3C 标准（2.2）。来源：W3C WAI。**常见误用**：当合规事后补丁而非内建。**地域**：WCAG 全球；中国另有信通院/国标差异。 (evidence: [T04-S015])

#### tier-2（周边 16）

13. **Conceptual model 概念模型** — `tier-2`。设计师传给用户的系统运作图景。来源：Norman。 (evidence: [T04-S001])
14. **Mapping / Feedback / Constraint** — `tier-2`。控件↔效果对应 / 即时回应 / 限制错误。来源：Norman。 (evidence: [T04-S001])
15. **Goal-directed design 目标导向设计** — `tier-2`。以用户目标(非任务)为中心。来源：Cooper。 (evidence: [T04-S006])
16. **Continuous discovery / OST** — `tier-2`。周节奏用户接触 + opportunity solution tree。来源：[primary] Torres (CDH 2021)。 (evidence: [T04-S010])
17. **Product discovery（discovery vs delivery）** — `tier-2`。先验证该不该做。来源：Cagan (Inspired)。 (evidence: [T04-S007])
18. **Design token 设计令牌** — `tier-2`。设计决策的命名变量。来源：Nathan Curtis / W3C Design Tokens CG。**演化**：从 Salesforce 实践 → W3C 标准化中。 (evidence: [T04-S012, T04-S015])
19. **Atomic design 原子设计** — `tier-2`。atoms→molecules→organisms→templates→pages。来源：Frost。 (evidence: [T04-S012])
20. **North-star metric / HEART** — `tier-2`。单一核心价值指标 / Google 体验度量框架(Happiness-Engagement-Adoption-Retention-Task success)。来源：产品/增长共识 / Google (Rodden et al)。 (evidence: [T04-S024])
21. **Jakob's Law** — `tier-2`。用户带着别处养成的习惯来 → 遵循惯例。来源：Nielsen。 (evidence: [T04-S018, T04-S004])
22. **Hick's Law / Fitts's Law / Miller 7±2** — `tier-2`。决策时间随选项增 / 目标获取时间与距离尺寸 / 工作记忆容量(常被误用为「每组≤7」)。来源：Hick-Hyman / Fitts / Miller（心理学经典）。**常见误用**：把 7±2 当导航项数硬规则。 (evidence: [T04-S018, T04-S014])
23. **Progressive disclosure 渐进披露** — `tier-2`。按需分步暴露复杂度。来源：Nielsen/HCI 实践。 (evidence: [T04-S021])
24. **Dark / Deceptive pattern 暗黑模式** — `tier-2`。诱导用户违背意愿的设计。来源：[primary] Harry Brignull (2010, deceptive.design)。**时效**：欧盟/FTC 立法监管中。 (evidence: [T04-S019])
25. **Usability testing / Think-aloud / Discount usability** — `tier-2`。出声思考观察 + 低成本快测。来源：Krug / Nielsen。 (evidence: [T04-S003, T04-S005])
26. **Card sorting / Tree testing / Affinity mapping** — `tier-2`。IA 研究方法。来源：IA/UX 研究实践。 (evidence: [T04-S021])
27. **SUS / NPS / standardized metrics** — `tier-2`。System Usability Scale(均值≈68) / Net Promoter Score。来源：[primary] John Brooke (SUS, 1986) / Reichheld (NPS)。**caveat**：SUS 68 是业内基准非铁律。 (evidence: [T04-S020])
28. **Mobile First / Responsive / Red routes / Happy path / Edge case / Empty state / Design debt** — `tier-2`（实践词族）。Mobile First 来源：[primary] Luke Wroblewski (2009)。 (evidence: [T04-S013, T04-S021])

## Phase 2 提炼提示

### 反复出现 ≥ 3 个 canon 都讨论的核心 idea（候选心智模型）

- **「You are not the user」/ 行为 > 自陈** 出现于: Krug DMMT (T04-S003) / Hall Just Enough Research (T04-S011) / Norman DOET (T04-S001) / NN/g 研究方法 (T04-S021) → **候选心智模型 #1**（最强）：设计决策必须基于观察到的行为，而非设计师/利益相关者的意见或用户的口头偏好。
- **降低认知负荷 / 别让用户思考 / 识别而非回忆** 出现于: Krug (T04-S003) / Nielsen 启发式#6 (T04-S004) / Weinschenk 100 Things (T04-S014) / Laws of UX-Hick (T04-S018) → **候选心智模型 #2**：好设计让正确操作显而易见，把思考成本从用户身上移走。
- **问题空间 vs 方案空间 / 先做对的事再把事做对** 出现于: Double Diamond (T04-S023) / Design Thinking (T04-S016) / Cagan discovery (T04-S007) / Torres OST (T04-S010) → **候选心智模型 #3**：发散-收敛节奏，先确保解决对的问题。
- **outcome > output / 服务产品成功而非个人作品集** 出现于: Cagan Inspired (T04-S007) / Torres CDH (T04-S010) / Gothelf Lean UX (T04-S009) → **候选心智模型 #4**：designer 的价值在用户与商业结果，不在交付物数量或视觉炫技。
- **一致性 / 遵循惯例（Jakob's Law）vs 适度创新** 出现于: Nielsen 启发式#4 (T04-S004) / Jakob's Law (T04-S018) / Atomic Design 复用 (T04-S012) → **候选心智模型 #5**：默认遵循用户已有心智模型与平台惯例，破规范需有充分理由。
- **迭代验证 / 小批量持续测试** 出现于: Nielsen 5-users (T04-S005) / Knapp Sprint (T04-S008) / Torres weekly touchpoints (T04-S010) / Lean UX 假设 (T04-S009) → **候选心智模型 #6**：用低成本快速循环逼近真相，而非一次性大投入。

- **认知心理/人因派**: 奠基 = Don Norman (DOET, T04-S001) ← Gibson 生态心理学。当前代表 = Susan Weinschenk (T04-S014) / Kathryn Whitenton (NN/g)。**关联 Track 01**: Norman, Weinschenk。
- **可用性工程派**: 奠基 = Jakob Nielsen (10 heuristics + 5 users, T04-S004/S005) / Steve Krug (T04-S003)。当前代表 = NN/g 团队 (T04-S021) / Jared Spool。**关联 Track 01**: Nielsen, Krug, Spool。
- **交互设计/目标导向派**: 奠基 = Alan Cooper (About Face / Inmates, T04-S006)。当前代表 = Kim Goodwin。**关联 Track 01**: Cooper, Goodwin。
- **设计思维派**: 奠基 = IDEO David Kelley / Tim Brown (HBR 2008, T04-S016) / Stanford d.school。**关联 Track 01**: Kelley, Brown。
- **精益/持续探索派**: 奠基 = Jeff Gothelf Lean UX (T04-S009) / Teresa Torres continuous discovery (T04-S010) / Marty Cagan product discovery (T04-S007)。**关联 Track 01**: Gothelf, Torres, Cagan。
- **设计系统/系统化派**: 奠基 = Brad Frost Atomic Design (T04-S012) / Nathan Curtis design tokens / Alla Kholmatova。**关联 Track 01**: Frost, Curtis。
- **视觉/UI craft 派**: 奠基 = Adam Wathan & Steve Schoger Refactoring UI (T04-S012) ← Dieter Rams "less but better" 工业设计遗产。**关联 Track 01**: Wathan, Schoger。

**主要分歧（still-debated within the canon）**:
1. **Design Thinking 是真方法还是咨询话术** — Tim Brown/Liedtka (辩护, T04-S016/S026) ⇄ Norman & Natasha Jen ("Design Thinking is Bullshit") 批其便利贴空洞化。 (evidence: [T04-S016, T04-S026])
2. **「5 个用户」是经验法则还是被滥用的伪铁律** — Nielsen 原模型 (T04-S005) ⇄ Jared Spool 等批语境局限。canon 内部对样本量没有共识。 (evidence: [T04-S005])
3. **设计系统提效 vs 扼杀创意** — Frost/系统派 ⇄ 视觉 craft 派担心同质化。 (evidence: [T04-S012])
4. **user-centered/research vs business/ship / craft vs data** — 第一性张力，canon 两侧都有奠基文（Norman/Hall 研究侧 ⇄ Cagan/Gothelf 商业交付侧），非可调和。 (evidence: [T04-S001, T04-S007])
5. **UX vs UI vs "product designer" 职称泛滥与「designer 该不该 code」** — 无 canonical 定论，是活跃职业争论。

### 核心概念 → 候选 playbook（判断方式）

- **affordance/signifier 不清** → 「如果用户不知道某元素可点/怎么操作，问题是缺 signifier 而非缺功能，先补可感知线索」。 (evidence: [T04-S001])
- **stakeholder 凭意见争论方案** → 「如果争执基于意见而非数据/观察，则做一次折扣可用性测试或拉行为数据，'opinions are not data'」。 (evidence: [T04-S003, T04-S005])
- **想加新功能** → 「先问这解决用户哪个 job（JTBD）/ 是 discovery 验证过的 opportunity 吗，否则警惕 feature creep」。 (evidence: [T04-S017, T04-S007])
- **流程一开始就纠结视觉** → 「如果还在问题空间(Discover/Define)就做 hi-fi，停下回到 lo-fi/research，先确认解决对的问题」。 (evidence: [T04-S023])
- **要不要破平台惯例** → 「默认遵循 Jakob's Law/惯例；破规范前确认收益 > 用户重新学习成本」。 (evidence: [T04-S018])
- **a11y 被排到最后** → 「无障碍是内建约束不是事后补丁，设计阶段就按 WCAG 检查对比度/键盘/语义」。 (evidence: [T04-S015])
- **诱导性设计请求（默认勾选/难取消）** → 「识别为 dark pattern，伦理+监管风险，拒绝或上报」。 (evidence: [T04-S019])

  1. **中英失衡**: 一手 canon ≈100% 英文。中文世界有庞大实践社群（腾讯 CDC / 阿里 UED / 字节 / 蓝湖 / 优设 / 站酷），但独立中文一手 canon 极少，多为译著/二手转述，部分沉淀在黑名单平台（知乎/公众号），未纳入本表。synthesis 引用中文源时须标 surrogate/转述。
