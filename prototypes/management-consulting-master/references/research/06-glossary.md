### 行业「外行破绽」高亮（最容易暴露的）

| 误用 | 内行实际意思 | 出现频率 |
|------|------------|---------|
| 把 framework 当「答案」直接套出来交差 | framework 只是脚手架，必须推出 so-what 才有价值 | 极高 |
| 罗列一堆发现，答不出 so-what | 每个分析都要逼问「所以对决策意味着什么」 | 极高 |
| 拿 SWOT 当高级战略分析 | 业内视 SWOT 为入门/无洞见，资深人很少单用 | 高 |
| 混淆 strategy 与 plan/计划 | strategy=取舍与定位 (做什么不做什么)，不是行动清单 | 高 |

## 重点术语详解（Tier 1 + 关键 Tier 2，带一手定义源 + 外行/内行差别）

### 1. MECE — 相互独立、完全穷尽
- **Type**: term + acronym (Mutually Exclusive, Collectively Exhaustive)
- **Insider def**: 拆问题/分桶的黄金律——子项之间不重叠 (ME)、合起来不遗漏 (CE)。是 issue tree、deck 结构、storyline 的底层语法。(evidence: [T06-S004, T06-S007])
- **Outsider def**: 把一团乱的东西分成「既不打架、又没漏掉」的几堆。
- **Etymology**: Barbara Minto 在 McKinsey (1960s 末) 提出，支撑她的 Pyramid Principle；Minto 自承理念可追到 Aristotle。(evidence: [T06-S004])
- **outsider-tell**:
  - `usage_tell`: 外行拆出来的桶互相重叠 (如把客户分「大客户 / 老客户 / VIP」——一个人可同时属三类) 还自称「全面」。(evidence: [T06-S004])
  - `pronunciation_tell`: 读「me-see」/「M-E-C-E」均可，但**不知道这个缩写**本身就是露馅。
- **disputed/批评**: MECE 被批①不排除冗余/无关项 (CE 不保证「无多余」)②强行 MECE 会过度限制答案空间，有时冗余反而必要。资深人知道 MECE 是工具不是教条。(evidence: [T06-S004])
- **关联**: issue tree / Pyramid Principle / 80-20

### 2. issue tree / logic tree — 问题树 / 逻辑树
- **Type**: term
- **Insider def**: 把核心问题逐层 MECE 拆成可独立验证的子问题，每个叶子能 yes/no 或量化检验。是 hypothesis-driven 解题的骨架。(evidence: [T06-S009, T06-S007])
- **outsider-tell**: `usage_tell` 外行把它做成发散 mind map，缺「每支都指向一个可验证假设」的纪律。
- **关联**: MECE / hypothesis-driven / driver tree

### 3. hypothesis-driven (假设驱动 / answer-first / Day-1 answer)
- **Type**: term
- **Insider def**: 项目一开始就基于经验立**可证伪**的初步答案 (Day-1 answer)，再定向找证据去 kill 或 keep，避免穷举。资深人偏 top-down 假设以减少 boil-the-ocean。(evidence: [T06-S009])
- **Outsider def**: 先大胆猜结论，再用数据快速验证/推翻——边做边修。
- **outsider-tell**:
  - `semantic_tell`: 外行听成「先有答案再硬凑支持证据」=confirmation bias。内行的纪律恰恰是「随时准备 kill 自己的假设」。这是本行最深的张力 (intake warning #5)。
  - `usage_tell`: 另一类外行走反面：「没数据前不许有观点」，于是 boil the ocean。
- **disputed**: 假设驱动的纪律 (敢 kill) vs 假设驱动的滥用 (只找支持证据) —— 同一方法的两面，反例节不可软化成纯优点。(evidence: [T06-S009])
- **关联**: Day-1 answer / so-what / boil the ocean

### 4. so what — 所以呢 / 洞见提炼
- **Type**: term + register marker
- **Insider def**: 从「数据/现象 (what)」逼出「对客户决策意味着什么 (so-what)」的那一跳。**会不会说 so-what 是区分咨询师与分析员的分水岭**。(evidence: [T06-S009])
- **outsider-tell**: `semantic_tell` 外行把「我发现 X 下降了 12%」当结论交差；内行追问「so what?→所以要砍掉该 SKU/重配预算」。**答不出 so-what 是外行一开口最大的破绽**。
- **关联**: action title / hypothesis-driven / Pyramid Principle

### 5. Pyramid Principle (Minto 金字塔原理) + SCQA
- **Type**: term (Pyramid) + acronym (SCQA)
- **Insider def**: Minto 的沟通法——**结论先行**，下面分组挂 3–4 个论据，自上而下、executive-friendly；开篇用 **SCQA (Situation-Complication-Question-Answer)** 制造「为什么现在要解决」的钩子。思考是 bottom-up (从发现到结论)，但**表达永远 top-down**。(evidence: [T06-S015, T06-S018])
- **Etymology**: Barbara Minto《The Pyramid Principle: Logic in Writing and Thinking》(1978)，McKinsey 沟通圣经。(evidence: [T06-S018])
- **outsider-tell**: `usage_tell` 外行按时间/发现顺序讲，把结论埋最后；开头堆背景没有 complication，引不出 question。
- **关联**: SCQA / action title / storyline / so-what

### 6. ghost deck / storyline / action title — 鬼影稿 / 故事线 / 行动标题
- **Type**: term ×3
- **Insider def**:
  - **ghost deck (又名 shell/skeleton deck)**: ~20% 完成度的骨架——只写好 action title + 占位空白 exhibit，先和团队/客户对齐**故事线**再填数据，最小化返工。(evidence: [T06-S015])
  - **storyline**: deck 背后由一句句 action title 串成的论证逻辑链——deck 是「一条论证」不是「页面集合」。
  - **action title**: 幻灯片标题写成**完整结论句**而非主题词，一页一条信息 (one message per slide)。三巨头通用规则。(evidence: [T06-S015])
- **outsider-tell**: `usage_tell` ①外行一上来做精美成品页，方向一改全废 (=没用 ghost deck)；②标题写「收入分析」(主题) 而非「收入因低端机型萎缩下滑 12%」(结论)。
- **关联**: deck / Pyramid Principle / exec sum

### 7. 80/20 (Pareto) & boil the ocean
- **Type**: term (80/20) + term-贬义 (boil the ocean)
- **Insider def**: 80/20=抓贡献大头的 20% 驱动因子，**主动放弃**长尾精度；与之对立的反模式叫 **boil the ocean**——范围失控、过度穷举、为不必要精度耗死团队。「let's not boil the ocean」比解释一长串快得多。(evidence: [T06-S010])
- **outsider-tell**: `register_tell`「boil the ocean」是**贬义/批评语**，外行不知道这是骂人话，真去把海煮干。
- **关联**: hypothesis-driven / 80-20 / scoping

### 8. leverage (model) / utilization / bench / up-or-out — 杠杆模型 / 稼动率 / 板凳 / 非升即走
- **Type**: term ×4 (行业结构黑话)
- **Insider def**:
  - **leverage model**: 1 个 partner 带多层 EM/associate/analyst 的金字塔；leverage=客户单价与人力成本的差，差越大毛利越高。业内俗称 **finders / minders / grinders** (找活的 / 管活的 / 干活的)。(evidence: [T06-S016, T06-S019])
  - **utilization**: 可计费时间占比，~75%+ 才盈利。(evidence: [T06-S016])
  - **bench**: 没派到项目上的待命状态 (久了焦虑、影响 utilization)。
  - **up-or-out**: 到点没晋升就得走的制度。
- **outsider-tell**: `semantic_tell`「leverage」对外行 = 金融杠杆/负债；咨询里指**人力金字塔配比/毛利结构**。
- **关联**: utilization / partner-track / up-or-out

### 9. case interview / PEI / fit / market sizing / exit opps — 招聘生态
- **Type**: term ×5
- **Insider def**:
  - **case interview**: 现场拆解一个商业问题，考 structure (MECE)+量化 (market sizing)+so-what。(evidence: [T06-S008])
  - **PEI (Personal Experience Interview)**: McKinsey 的行为面，**单个 story 深挖 10–25 个追问**，考 Connection/Drive/Leadership/Growth；与 case 权重相当，很多人栽在 PEI 而非 case。(evidence: [T06-S008])
  - **fit interview**: PEI 的通称 (= CV/行为面)。
  - **market sizing**: 估算题，拆「人口×渗透率×单价」结构化推。
  - **exit opps**: 咨询跳出去的去向 (PE/VC/corp dev/战略/创业)。
- **outsider-tell**: `register_tell` ①外行面试里大谈 exit opps/薪资——内部都懂为 exit 来，**说出口=减分**；②把 case 当普通行为面，不 structure。(evidence: [T06-S008])
- **关联**: networking / ding / up-or-out

### 10. Frameworks（Tier 2，既是术语也是工具，附「双刃」批判）
- **Porter's Five Forces** (五力): 新进入者/替代品/买方议价/卖方议价/同业竞争决定行业盈利结构；战略=找力较弱的位置或筑防御。Porter《How Competitive Forces Shape Strategy》HBR 1979/重刊 2008。(evidence: [T06-S001])
- **value chain**: Porter《Competitive Advantage》1985，把企业拆 primary (进料/生产/出货/营销/服务) + support (采购/技术/HR/基建) 活动找优势来源。(evidence: [T06-S022])
- **BCG growth-share matrix**: Henderson《The Product Portfolio》(Perspectives, 1970)，star/cash cow/question mark/dog 管业务组合现金流。又名 Boston Box。(evidence: [T06-S002, T06-S005])
- **7S (McKinsey)**: Strategy/Structure/Systems/Style/Staff/Skills/Shared values，源自 Waterman&Peters《Structure Is Not Organization》(Business Horizons, 1980)。(evidence: [T06-S006])
- **experience curve / 3C / 4P / Ansoff / PESTEL**: 标准化框架，定义稳定，略。
- **SWOT — 被业内视为入门 + 被批**: `Type: framework, disputed`。内行普遍视 SWOT 为低阶/无洞见工具——「vanilla SWOT 不产生能导出好战略的洞见」「常被用来充门面或填补理解空白」，资深人很少单独用。HBR 甚至教「倒着做 SWOT」才有用。**反例节须诚实保留此批判，不软化成『咨询都很科学』**。(evidence: [T06-S020])
- **frameworks 双刃性 (intake warning #4)**: framework 是脚手架不是答案；「套框架剧场 (frameworks-as-theater)」是行业自我批判 (Mintzberg 批 planning school；Matthew Stewart《The Management Myth》)。外行最大破绽就是把 framework 当答案而推不出 insight。

### 11. 项目/合规黑话 (Tier 2，快速定义)
engagement/case/study (项目)、SOW/scope of work (范围书)、RFP (招标)、BD (拓客)、SteerCo/steering committee (客户高管委员会)、stakeholder (利益相关方)、deliverable/end product (交付物)、workstream (工作流)、deck/exec sum (幻灯片/执行摘要)、expert call (专家电话)、CDD/commercial DD (商业尽调：市场/竞争/客户/增长可持续，PE 投前)、implementation (落地)、readout (阶段汇报)。(evidence: [T06-S009, T06-S019])

### 12. 标准 / 认证 / 合规
- **CMC (Certified Management Consultant)** — `certification`: ICMCI/CMC-Global 颁发，全球唯一被广泛承认的管理咨询个人认证。能力框架 v4.2 含 **8 项能力、3 大主题** (Business / Technical / Values&Behaviour)，并要求承诺 ICMCI Code of Conduct (含法律/社会/生态/职业操守)。但**注意：MBB 招聘几乎不看 CMC**——它在独立顾问/中小所/政府采购里更有分量。(evidence: [T06-S012, T06-S013]) `Decay: medium`
- **MCA (Management Consultancies Association, UK)** + **Consulting Excellence (2014)**: 英国行业协会，会员约占英国咨询业 60%；Consulting Excellence 是自律方案 (伦理行为/客户价值/职业发展三支柱)；并推 **Chartered Management Consultant (ChMC, 2021)** 新特许认证。(evidence: [T06-S014])
- **伦理/利益冲突/独立性 + 保密 (NDA)**: 咨询无强制法律监管，软约束靠①协会 Code of Conduct ②客户合同 NDA/保密 ③利益冲突回避 (尤其同时服务竞品/买卖双方)。`Regulation 维度 = N/A (无强监管法律)`。
- **变化历史**: CMC 1987 起源→框架迭代至 v4.2；ChMC 2021 新增。

### 13. GenAI 术语新增层（decay: HIGH）
- **Insider def**: 2023–2025 起 GenAI 进入日常语汇。McKinsey 内部 **Lilli** (检索 10 万+ 内部文档、起草 deck/research，~72% 员工使用，号称省 30% research 时间)；BCG **Deckster** (自动美化 PPT)。(evidence: [T06-S003, T06-S021])
- **行业张力**: 有观察称到 2025 GenAI 可完成初级分析师 ~80% 的 research+做 deck 工作 → 冲击金字塔底层 (intake (g))；但工具也会幻觉 (Lilli 曾引用不存在的法规)，仍需人工核验。**此为 master skill 时效衰减最快的一节。**(evidence: [T06-S021])

## Phase 2 提炼提示

### 「行业表达 DNA」直接素材（喂 Phase 2.5 expression-DNA）

**高频黑话 top 10 (Tier 1)**:
1. MECE 2. issue tree 3. hypothesis-driven (Day-1 answer / answer first) 4. **so what** 5. Pyramid Principle / SCQA 6. 80/20 + boil the ocean 7. action title / storyline / ghost deck 8. top-down vs bottom-up 9. leverage model / utilization 10. market sizing
(evidence: [T06-S009, T06-S010, T06-S015])

**外行破绽 top 10 (insider-only spotting tells)**:
1. 把 framework 当答案直接套出来，推不出 insight
2. 罗列发现答不出 **so what**
3. 拿 SWOT 当高级战略分析
4. 混淆 strategy 与 plan/计划 (战略=取舍定位，非行动清单)
5. 不知道 MECE / 拆桶重叠又漏项
6. 把 hypothesis-driven 理解成「先有结论硬凑证据」(confirmation bias)
7. 真的去 boil the ocean (不会 80/20)
8. 把 leverage 听成金融杠杆/负债
9. action title 写成主题词而非结论句
10. 求职面试里大谈 exit opps/薪资 (register 错)
(evidence: [T06-S004, T06-S009, T06-S020])

**行业拒绝/自嘲的厂商话术 top 5 (拒绝 → 反映价值观+反模式)**:
- **「借你的表告诉你时间」(borrow your watch to tell you the time)** —— 行业最经典自嘲，讽刺「拿客户自己的数据反卖回去还收费」；衍生版「偷你的表」。(evidence: [T06-S017])
- **buzzword bingo 被嘲清单**: **synergy** (协同)、**paradigm shift** (范式转移)、**low-hanging fruit** (低垂果实)、**leverage** (当动词滥用)、**bandwidth / drill down / move the needle / circle back / boil the ocean (滥用)** —— 内行自己开「buzzword bingo」嘲笑。88% B2B 买家因 buzzword 失去信任。(evidence: [T06-S010, T06-S011])
- **「frameworks-as-theater / 套框架剧场」** —— 把 2x2 和五力当表演道具而非真分析 (Mintzberg / Matthew Stewart《The Management Myth》批判)。
- **SWOT 当深度分析** —— 内行视为入门级，单用即露怯。(evidence: [T06-S020])
- **区分「真术语 vs 被嘲 buzzword」**: 真术语 (MECE/so-what/issue tree/Day-1 answer) 是有操作内核的；被嘲 buzzword (synergy/paradigm shift/low-hanging fruit) 是空心装饰词。**资深人用前者、嘲后者**——这条分界本身就是表达 DNA。

### 「智识谱系」线索（喂 Phase 2.7）
- 框架演进暗示范式：Porter **定位学派** (Five Forces 1979 → value chain 1985，行业结构决定论) ⇄ 后来的资源基础观/蓝海/颠覆式创新对它的反动；BCG 经验曲线+增长份额矩阵 (1970) 代表「现金流组合」范式。
- 沟通范式：Minto Pyramid (1970s) 把「结论先行 + MECE」固化为全行业母语——这是方法论上的「奠基范式」，至今未被取代 (decay low)。
- GenAI 层 (2023–) 是正在发生的范式扰动：冲击 leverage model 底层 (analyst 工作被自动化)。

### 「时效性」信号（喂 Phase 2.8 诚实边界）
- **decay HIGH 的只有 GenAI 术语层** (Lilli/Deckster/AI-on-analyst-work)——12 月内必变。
- **CMC/ChMC 认证** decay medium (框架版本会迭代，ChMC 2021 仍新)。
- **Tier 1 方法论黑话全部 decay low**——MECE/so-what/Pyramid/issue tree 稳定 40+ 年，是 master skill 最硬的内核。
- 中英失衡：术语一手定义几乎全英文，中文多为翻译/二手 (intake warning #1)，诚实边界须明示。

### 「工作流变化触发」种子（喂 wave 3 Track 03）
